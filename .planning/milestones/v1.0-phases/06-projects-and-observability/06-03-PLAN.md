  <action>
1. Extend `DashboardService` with query analytics methods (ALL must include `WHERE tenant_id = ?` — Pitfall 4):

   **getQueryAnalytics(tenantId, startDate, endDate)** — returns Promise with:
   - totalQueries: COUNT(*) from query_log WHERE tenant_id AND date range
   - avgResponseTime: AVG(response_time_ms) from query_log WHERE tenant_id AND date range
   - failedRate: COUNT(WHERE failed_retrieval=true) / totalQueries as percentage
   - lowConfRate: COUNT(WHERE confidence_score IS NOT NULL AND confidence_score < 0.5) / totalQueries as percentage
   - topQueries: SELECT query, COUNT(*) as count, AVG(confidence_score) as avg_confidence FROM query_log WHERE tenant_id AND date range GROUP BY query ORDER BY count DESC LIMIT 10
   - trend: Daily query counts grouped by date (reuse getDailyCount pattern but on query_log table with tenant_id filter)
   Run all 6 queries in parallel with Promise.all (follow existing getStats pattern).

   **getFeedbackAnalytics(tenantId, startDate, endDate)** — returns Promise with:
   - satisfactionRate: COUNT(WHERE thumbup=true) / COUNT(*) * 100 from answer_feedback WHERE tenant_id AND date range
   - totalFeedback: COUNT(*) from answer_feedback WHERE tenant_id AND date range
   - zeroResultRate: COUNT(WHERE query IN (SELECT query FROM query_log WHERE failed_retrieval=true AND tenant_id)) / totalFeedback — or simpler: join with query_log on matching query+tenant
   - worstDatasets: For each unique source_id in answer_feedback WHERE source='search' AND tenant_id, calculate thumbup ratio. Return bottom 5 dataset names (join with knowledgebase table on source_id).
   - trend: Daily feedback counts and daily satisfaction rate grouped by date
   - negativeFeedback: SELECT id, query, answer (truncated to 200 chars), trace_id, created_at FROM answer_feedback WHERE thumbup=false AND tenant_id AND date range ORDER BY created_at DESC LIMIT 20

   Add a helper `getLangfuseTraceUrl(traceId: string): string` that constructs `${config.langfuse.baseUrl.replace(/\/$/, '')}/trace/${traceId}` (Pitfall 6: strip trailing slash).

2. Add controller methods:
   - `getQueryAnalytics(req, res)`: Extract tenantId from req (via requireTenant), startDate/endDate from query params. Call dashboardService.getQueryAnalytics(tenantId, startDate, endDate). Return 200.
   - `getFeedbackAnalytics(req, res)`: Same pattern. Call dashboardService.getFeedbackAnalytics(tenantId, startDate, endDate). Return 200 with langfuseBaseUrl included in response for frontend deep-link construction.

3. Add routes (follow existing pattern):
   - `GET /analytics/queries` — requireAuth, requireRole('admin', 'super-admin'), controller.getQueryAnalytics.bind(controller)
   - `GET /analytics/feedback` — requireAuth, requireRole('admin', 'super-admin'), controller.getFeedbackAnalytics.bind(controller)

   Note: CONTEXT.md says "admin + super-admin have analytics by default. Leaders get analytics access when granted by admin." For initial implementation, gate on admin/super-admin role. Leader access gating via CASL ability can be added as enhancement.
