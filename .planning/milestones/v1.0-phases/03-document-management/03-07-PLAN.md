<objective>
Rewrite the cron parsing scheduler to use dataset-aware FIFO sequencing instead of the current global LIMIT 50 approach that processes all datasets at once.

Purpose: UAT Test 14 reported that cron parsing should be "queue 1st in 1st out" per dataset, not a global batch. The current implementation queries all pending docs across all datasets simultaneously and pushes up to 50 into the Redis Stream. This must be changed to group by kb_id, process one dataset batch at a time in FIFO order, with a per-dataset document limit.

Output: Updated runParsingSchedule method in cron.service.ts with dataset-aware sequencing.
</objective>

   // Query pending documents ordered by dataset, then by creation time (FIFO)
   const pendingDocs = await db('document')

       .where('run', '1')

   ```typescript
   // Group pending documents by dataset for sequential dataset processing
   const datasetGroups = new Map<string, typeof pendingDocs>()
   for (const doc of pendingDocs) {
       const group = datasetGroups.get(doc.kb_id) || []
       group.push(doc)
       datasetGroups.set(doc.kb_id, group)
   }
   ```
