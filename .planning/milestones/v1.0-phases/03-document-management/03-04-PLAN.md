**1. Add bulkUpdateMetadata method** to `RagService`:
```typescript
/**
 * @description Bulk update metadata_tags on multiple datasets
 * @param datasetIds - Array of dataset UUIDs to update
 * @param metadataTags - Key-value pairs to set/merge into parser_config.metadata_tags
 * @param mode - 'merge' adds to existing metadata_tags, 'overwrite' replaces metadata_tags entirely
 * @param tenantId - Tenant for isolation
 */
async bulkUpdateMetadata(datasetIds: string[], metadataTags: Record<string, string>, mode: 'merge' | 'overwrite', tenantId: string): Promise<void>
```
- For 'merge': use Knex raw `jsonb_set(parser_config, '{metadata_tags}', COALESCE(parser_config->'metadata_tags', '{}') || ?)` to merge new tags into existing metadata_tags
- For 'overwrite': use `jsonb_set(parser_config, '{metadata_tags}', ?)` to replace metadata_tags entirely
- Use `whereIn('id', datasetIds).andWhere('tenant_id', tenantId)` for batch + ABAC isolation
- CRITICAL: Store in `parser_config.metadata_tags`, NOT `parser_config.metadata` (that key is reserved for auto-extraction schema)

**2. Add getTagAggregations method** to `RagSearchService`:
```typescript
/**
 * @description Query OpenSearch for unique tag keys and top values from accessible datasets
 * @param tenantId - Tenant filter
 * @param datasetIds - Optional dataset ID scope
 * @returns Tag keys with their top values
 */
async getTagAggregations(tenantId: string, datasetIds?: string[]): Promise<{ key: string; values: string[] }[]>
```

- Build OpenSearch aggregation: `{ size: 0, query: { bool: { filter: [...] } }, aggs: { tag_keys: { terms: { field: 'tag_kwd', size: 50 } } } }`
- Filter by tenant + optional dataset IDs (strip hyphens per Pitfall 6)
- Return `{ key, values }[]`
