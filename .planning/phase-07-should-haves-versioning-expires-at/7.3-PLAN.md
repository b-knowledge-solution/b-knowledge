<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated browser -> Socket.IO connection | Client receives server-driven invalidation signals and must not be able to mutate catalog state directly through arbitrary socket payloads. |
| runtime catalog provider -> permission-gated UI | Live metadata refresh changes FE gating decisions and must stay fail-closed under drift or partial refresh. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-3-01 | E | runtime `useHasPermission()` | mitigate | Keep unknown-key behavior fail-closed and rebuild the map only from authenticated catalog responses plus the generated snapshot seed. |
| T-07-3-02 | D | catalog polling | mitigate | Use Socket.IO as the fast path and bound polling to minutes on `queryKeys.permissions.catalog()` so the fallback does not create an aggressive refresh loop. |
| T-07-3-03 | T | socket event handling | mitigate | Invalidate/refetch on a canonical server event constant; do not trust arbitrary payload data to mutate the in-memory catalog directly. |
| T-07-3-04 | I | FE contract drift | mitigate | Align FE `PermissionCatalogResult` and API wrapper to the backend `{ version, permissions }` contract and cover refresh behavior in FE regression tests. |
</threat_model>
