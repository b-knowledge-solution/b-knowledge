    Exports a default `PermissionMatrix` component. Props: none (self-contained) OR `{ roles: string[] }` if you need to parameterize.

    Implementation:
    1. Import `PERMISSION_KEYS` from `@/constants/permission-keys` and convert the const map to an array of keys. Group keys by the substring before the first `.` — produce `Record<prefix, string[]>`. Section header labels come from i18n keys `permissions.admin.matrix.sections.{prefix}` with the raw prefix as fallback.
    2. Import the roles list from `@/constants/roles` (confirm exact export name — expected `ROLES` or similar). Build column header list. Do NOT hardcode any role strings.
    3. For each role column, call `useRolePermissions(role)`. This yields a `Set<string>` of that role's current permission keys.
    4. Dirty state: `const [dirty, setDirty] = useState<Map<string, Set<string>>>(new Map())`. Map of role → full replacement set. On any cell toggle, clone the source set (server snapshot for that role), apply the toggle, store in `dirty`.
    5. Render: outer `<Card>` with header `permissions.admin.matrix.title`; body = grouped table. Each group has a `<h3>` with the section label and a sub-table: row = permission key, columns = roles, cell = `<Checkbox>` bound to `(dirty.get(role) ?? serverKeys).has(key)`. Dark-mode classes on every surface: `dark:bg-slate-800 dark:text-white` etc.
    6. Sticky footer: render only when `dirty.size > 0`. Contains "Save changes (N)" button + Cancel button. N = `dirty.size`.
    7. Save handler: instantiate `const mutation = useUpdateRolePermissions()`. For each `[role, keys]` in dirty, call `await mutation.mutateAsync({ role, permission_keys: Array.from(keys) })`. On all-success, clear dirty state and call `globalMessage.success(t('permissions.admin.sessionRefreshNotice'))`. On any error, `globalMessage.error(t('permissions.admin.saveError'))`.
    8. Cancel handler: `setDirty(new Map())`.
    9. JSDoc header comment on the component per CLAUDE.md.
