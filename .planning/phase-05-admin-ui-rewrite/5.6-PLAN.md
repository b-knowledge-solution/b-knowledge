<objective>
Build the new P5.6 "Effective Access" page per D-11 (scope addition). Adopt the simpler single-feature view strategy from research §6 to avoid the O(features × principals) fan-out problem.

Purpose: D-11 gives admins a user-centric answer to "who can do X?" without requiring them to open individual user detail pages. The user explicitly requested this framing.
Output: New page, route, sidebar entry, tests, i18n.

**Strategy (per research §6 + §12):**
- **Single-feature view**: Dropdown picks ONE feature (e.g. "knowledge_base.view"). Page makes ONE `whoCanDo(action, subject)` call for the selected feature. Result = list of users. Render as a table: rows = principals, single ✓ column.
- This avoids the 80×100 cartesian product. First version ships fast.
- Pagination/virtualization deferred unless admins hit scale issues.
- Drill-down: clicking a user row → `navigate('/iam/users/:id?tab=permissions')` (P5.2). No team drill-down in v1 because `whoCanDo` returns users only per research §6 service signature.
</objective>

    ```tsx
    /**
     * @description Read-mostly admin view: "Who can do X?" Single-feature selector + users-table powered by whoCanDo. Per 5-CONTEXT.md D-11.
     * @returns {JSX.Element} Effective Access page
     */
    export default function EffectiveAccessPage() {
      const { t } = useTranslation()
      const navigate = useNavigate()
      const [searchParams, setSearchParams] = useSearchParams()
      const selectedKey = searchParams.get('key') ?? PERMISSION_KEYS.KNOWLEDGE_BASE_VIEW

      // Map PERMISSION_KEYS dotted value -> (action, subject) for whoCanDo
      const { action, subject } = keyToActionSubject(selectedKey)
      const { data, isLoading } = useWhoCanDo(action, subject)

      return (
        <div className="p-6">
          <h1 className="text-2xl font-bold mb-4 dark:text-white">{t('permissions.admin.effectiveAccess.title')}</h1>
          <label className="block mb-2 dark:text-white">{t('permissions.admin.effectiveAccess.selectFeature')}</label>
          <select
            value={selectedKey}
            onChange={e => setSearchParams({ key: e.target.value })}
            className="border rounded p-2 mb-4 dark:bg-slate-800 dark:text-white dark:border-slate-700"
          >
            {Object.values(PERMISSION_KEYS).map(k => (
              <option key={k} value={k}>{k}</option>
            ))}
          </select>

          {isLoading && <div>{t('common.loading')}</div>}
          {data && data.users.length === 0 && (
            <div className="text-muted-foreground dark:text-slate-400">{t('permissions.admin.effectiveAccess.empty')}</div>
          )}
          {data && data.users.length > 0 && (
            <table className="w-full border dark:border-slate-700">
              <thead className="bg-gray-50 dark:bg-slate-700">
                <tr>
                  <th className="p-2 text-left dark:text-white">{t('permissions.admin.effectiveAccess.user')}</th>
                  <th className="p-2 text-left dark:text-white">{t('permissions.admin.effectiveAccess.email')}</th>
                  <th className="p-2 text-left dark:text-white">{t('permissions.admin.effectiveAccess.has')}</th>
                </tr>
              </thead>
              <tbody>
                {data.users.map(u => (
                  <tr
                    key={u.id}
                    onClick={() => navigate(`/iam/users/${u.id}?tab=permissions`)}
                    className="cursor-pointer hover:bg-gray-50 dark:hover:bg-slate-700 dark:text-white"
                  >
                    <td className="p-2">{u.display_name}</td>
                    <td className="p-2">{u.email}</td>
                    <td className="p-2">✓</td>
                  </tr>
                ))}
              </tbody>
            </table>
          )}
        </div>
      )
    }

    /** Maps a PERMISSION_KEYS dotted value like 'knowledge_base.view' to ({ action, subject }) for the whoCanDo call. */
    function keyToActionSubject(key: string): { action: string; subject: string } {
      const [prefixRaw, action = 'view'] = key.split('.')
      // Convert snake_case prefix to PascalCase subject: knowledge_base -> KnowledgeBase
      const subject = (prefixRaw ?? '').split('_').map(s => s.charAt(0).toUpperCase() + s.slice(1)).join('')
      return { action, subject }
    }
    ```
