# Phase 5 Plan 5.0b: Permissions Feature Scaffold Summary

## One-liner
Scaffolded `fe/src/features/permissions/` with a typed HTTP client + TanStack Query hooks covering all 10 Phase 3 permissions admin endpoints, installed shadcn Command primitive, and added 13 passing unit tests — unblocking parallel execution of Wave 1 (P5.1/P5.2/P5.3/P5.6).

## What Was Built

### Task 1 — shadcn Command primitive
- Ran `npx shadcn@latest add command` from `fe/` (declined `dialog.tsx` overwrite prompt).
- Added `fe/src/components/ui/command.tsx` and `cmdk` dep.
- `@radix-ui/react-popover` and `popover.tsx` were already present — skipped.
- **Commit:** `91b460d`

### Task 2 — Permissions feature module + tests
- **Types** (`permissions.types.ts`): `OverrideEffect`, `GranteeType`, `GrantResourceType` as constant-derived unions; `UserPermissionOverride`, `ResourceGrant`, `RolePermissions`, `WhoCanDoResult`, `PermissionCatalogResult`; request body shapes.
- **Raw HTTP client** (`permissionsApi.ts`): 10 endpoints — `getCatalog`, `whoCanDo`, `getRolePermissions`, `updateRolePermissions`, `getUserOverrides`, `createOverride`, `deleteOverride` (flat path), `getGrants`, `createGrant`, `deleteGrant`. Path literals pulled to `PERMISSIONS_API_BASE` + sub-path consts per no-hardcoded-strings rule. Also exports a `permissionsApi` namespace object.
- **Query hooks** (`permissionsQueries.ts`): `usePermissionCatalog`, `useRolePermissions`, `useUpdateRolePermissions`, `useUserOverrides`, `useCreateOverride`, `useDeleteOverride`, `useGrants`, `useCreateGrant`, `useDeleteGrant`, `useWhoCanDo`. Mutations invalidate narrowest matching cache key.
- **queryKeys.ts**: Added `permissions` namespace with `catalog`, `rolePermissions`, `userOverrides`, `grants`, `whoCanDo` factories.
- **Barrel** (`index.ts`): Re-exports api + queries + types.
- **Tests** (`permissionsApi.test.ts`): 13 vitest specs mocking `@/lib/api`. Covers all 10 endpoints + two `queryKeys.permissions` key shape assertions + explicit guard that `deleteOverride` never touches `/users/`.
- **vitest.shared.ts**: Added the test file to the `unitTestFiles` allowlist (required by existing pattern).
- **Commit:** `04b0bd2`
