- Rewrote all live backend `UserRole.SUPERADMIN` usages to `UserRole.SUPER_ADMIN`.
- Removed the dead `SUPERADMIN` and `MEMBER` keys from `UserRole` while preserving `TeamRole.MEMBER`.
- Added `scripts/check-legacy-roles.sh` and wired `npm run check:legacy-roles` in `package.json`.
- Fixed one frontend holdout in `PrincipalPicker` so the new guard passes across the repo.

## Decisions Made

- The guard explicitly targets `UserRole.SUPERADMIN`, `UserRole.MEMBER`, and bare `'superadmin'` aliases while leaving TeamRole contexts alone.
- The constant file is allowlisted for the bare-string scan where it documents removed aliases in JSDoc, avoiding a false positive from the cleanup rationale itself.

## Deviations from Plan

- The plan was backend-scoped, but the new repo-level guard exposed one frontend `UserRole.MEMBER` usage that also had to be cleaned up for the guard to be useful.

## Issues Encountered

- The first guard pass failed because the frontend still referenced the removed member alias.
- The bare-string check initially flagged the `roles.ts` JSDoc after the cleanup because the documentation mentioned `'superadmin'`; the guard was narrowed to exclude that intentional documentation site.
