<objective>
Add custom version label support to the version-as-dataset model so users can enter semantic version strings (e.g., "1.2.0", "Q1 Release") instead of only seeing auto-incrementing v{N} badges.

Purpose: UAT Test 2 reported that version display should support semantic versioning or custom labels, not just integer-based v{N}. The integer version_number is kept for pagerank boost; a new version_label text column provides display flexibility.

Output: New DB migration, updated service/API/schema, FE dialog with label input, VersionBadge with label-first rendering.
</objective>
