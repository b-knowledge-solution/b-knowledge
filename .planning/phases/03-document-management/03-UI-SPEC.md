## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, gap-1 |
| sm | 8px | Compact element spacing, gap-2, p-2 |
| md | 16px | Default element spacing, gap-4, p-4 |
| lg | 24px | Section padding, card body padding, gap-6 |
| xl | 32px | Layout gaps between major panels, gap-8 |
| 2xl | 48px | Major section breaks, py-12 empty states |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Split-view divider: 1px border-l (existing pattern from DocumentPreviewer)
- Touch targets: minimum 28px (h-7 w-7) for icon-only action buttons in table rows (existing pattern)
- Filter chip height: 28px (h-7) for inline tag filter chips below search bar

## Interaction Patterns

### Three Document Viewer Patterns

| Pattern | Container | Trigger | Layout |
|---------|-----------|---------|--------|
| Dataset split view | Full page | Click document name in DocumentTable | 50/50 split: preview left, chunk list right |
| Chat drawer | Sheet (side drawer) | Click citation tooltip in chat message | Full-width sheet with preview + highlight scroll |
| Search dialog | Dialog (modal) | Click search result card | Large dialog (max-w-5xl) with preview + highlight scroll |

All three patterns reuse the same `DocumentPreviewer` rendering component. Only the outer container differs.

### Version Upload Flow

1. User clicks kebab menu on document row in DocumentTable
2. User selects "Upload New Version" from dropdown menu
3. UploadNewVersionDialog opens with: file drop zone, optional change summary input, auto-parse toggle
4. On submit: new version dataset created, appears in dataset list with "Processing" badge
5. Dataset list shows flat list with version badges (v1, v2, v3) -- no grouping or collapsing

### Metadata Bulk Edit Flow

1. User selects multiple documents via checkboxes in DocumentTable
2. Bulk action bar appears with "Edit Tags" button
3. MetadataManageDialog opens in bulk mode with merge/overwrite toggle
4. User edits key-value pairs; on save, all selected documents updated

### Filter Chip Interaction

1. TagFilterChips renders available tag keys as outline badges below search bar
2. Clicking a chip opens a Popover with top values for that key (from OpenSearch aggregation)
3. Selecting a value activates the filter: chip changes to variant="secondary" with value shown
4. Active filters are additive (AND). Clicking X on active chip removes that filter
5. Filters respect ABAC -- only keys/values from accessible datasets appear

### Chunk CRUD on Detail Page

1. ChunkToolbar shows chunk count badge, search input, sort select, "Add Chunk" button
2. Click chunk card to select -- preview panel scrolls/highlights corresponding position
3. Inline edit: click edit icon on chunk card to enter edit mode (ChunkEditInline)
4. Delete: click delete icon on chunk card, confirm via existing confirm dialog pattern
5. Keywords and questions displayed as secondary badges below chunk text

## State Patterns

### Loading States

| Context | Pattern |
|---------|---------|
| Document table loading | Spinner centered in table area (existing pattern) |
| Chunk list loading | Spinner centered in chunk panel |
| Version upload in progress | "Processing" badge on new dataset row + Progress bar |
| Metadata save | Button disabled with spinner icon |
| Filter chip values loading | Popover shows small spinner while aggregation loads |

### Error States

| Context | Pattern |
|---------|---------|
| Parse failure | Destructive badge "Failed" on document row; clickable to open process log dialog |
| Version upload failure | Sonner toast with error message |
| Metadata save failure | Sonner toast with retry suggestion |
| Chunk load failure | Alert component inline in chunk panel |
| Filter aggregation failure | Filter chips hidden gracefully, no error shown to user |

### Optimistic Updates

| Action | Optimistic Behavior |
|--------|-------------------|
| Toggle auto-parse switch | Switch reflects new state immediately; reverts on API failure |
| Delete chunk | Chunk removed from list immediately; restored on API failure |
| Activate filter chip | Chip shows active state immediately; results refetch in background |
