**Step 1: Add ABAC-to-OpenSearch filter translator in ability.service.ts**

Add a new exported function `buildOpenSearchAbacFilters(policies: AbacPolicyRule[], userAttributes: Record<string, unknown>): Record<string, unknown>[]`:

This function translates ABAC policy conditions into OpenSearch bool/filter clauses:
- For each policy with `effect: 'allow'` and `action: 'read'` and `subject: 'Document'`:
  - Convert each condition key-value pair to an OpenSearch filter:
    - `{ department: 'clinical' }` becomes `{ term: { 'department': 'clinical' } }`
    - `{ department: { $in: ['clinical', 'research'] } }` becomes `{ terms: { 'department': ['clinical', 'research'] } }`
    - `{ tags: { $in: ['confidential'] } }` becomes `{ terms: { 'tags': ['confidential'] } }`
  - Wrap allow conditions in a `bool.should` (OR logic -- user matches ANY allow rule)
- For each policy with `effect: 'deny'`:
  - Convert to `bool.must_not` filters
- Return the combined filter array

Also add `buildAccessFilters(tenantId: string, abacFilters: Record<string, unknown>[]): Record<string, unknown>[]`:
```typescript
export function buildAccessFilters(
  tenantId: string,
  abacFilters: Record<string, unknown>[] = []
): Record<string, unknown>[] {
  return [
    { term: { tenant_id: tenantId } },  // Mandatory tenant isolation
    ...abacFilters,
  ]
}
```
