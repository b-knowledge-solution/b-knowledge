Define CASL types:
```typescript
import { AbilityBuilder, createMongoAbility, MongoAbility, RawRuleOf } from '@casl/ability'
import { redisService } from '@/shared/services/redis.service.js'
import { config } from '@/shared/config/index.js'
import { log } from '@/shared/services/logger.service.js'

type Actions = 'manage' | 'create' | 'read' | 'update' | 'delete'
type Subjects = 'Dataset' | 'Document' | 'ChatAssistant' | 'SearchApp' | 'User' | 'AuditLog' | 'Policy' | 'Org' | 'Project' | 'all'
export type AppAbility = MongoAbility<[Actions, Subjects]>
```

Interface for user context:
```typescript
export interface AbilityUserContext {
  id: string
  role: string
  is_superuser?: boolean | null
  current_org_id: string
  department?: string | null
  attributes?: Record<string, unknown>
}

export interface AbacPolicyRule {
  id: string
  effect: 'allow' | 'deny'
  action: string
  subject: string
  conditions: Record<string, unknown>
  description?: string
}
```

Build function `buildAbilityFor(user: AbilityUserContext, policies: AbacPolicyRule[] = []): AppAbility`:
- If `user.is_superuser === true` or `user.role === 'super-admin'`: `can('manage', 'all')`, return build()
- All authenticated users: `can('read', 'Dataset', { tenant_id: user.current_org_id })`, `can('read', 'Document', { tenant_id: user.current_org_id })`
- Admin: `can('manage', 'User', { tenant_id: user.current_org_id })`, `can('manage', 'Dataset', ...)`, `can('manage', 'Document', ...)`, `can('read', 'AuditLog', { tenant_id: user.current_org_id })`, `can('manage', 'ChatAssistant', ...)`, `can('manage', 'SearchApp', ...)`
- Leader: `can('create', 'Dataset', ...)`, `can('update', 'Dataset', ...)`, `can('delete', 'Dataset', ...)`, `can('manage', 'Document', ...)`, `can('manage', 'ChatAssistant', ...)`, `can('manage', 'SearchApp', ...)`
- User: only the base read permissions above
- Apply ABAC policies: iterate `policies`, for `deny` use `cannot()`, for `allow` use `can()`
- Return `build()`

Cache functions (ABILITY_CACHE_PREFIX = `'ability:'`):
- `cacheAbility(sessionId: string, ability: AppAbility): Promise<void>` -- serialize `ability.rules` to JSON, store with TTL from config session TTL (7 days = 604800 seconds)
- `loadCachedAbility(sessionId: string): Promise<AppAbility | null>` -- load from Redis, parse, return `createMongoAbility(rules)` or null
- `invalidateAbility(sessionId: string): Promise<void>` -- delete the key
- `invalidateAllAbilities(): Promise<void>` -- use redis SCAN with pattern `ability:*` and delete matches (for platform policy changes)

Export singleton: `export const abilityService = { buildAbilityFor, cacheAbility, loadCachedAbility, invalidateAbility, invalidateAllAbilities }`

All functions must have JSDoc per project conventions.

**Step 1: Create tenant.middleware.ts**

Create `be/src/shared/middleware/tenant.middleware.ts`:

```typescript
import { Request, Response, NextFunction } from 'express'
import { log } from '@/shared/services/logger.service.js'
```

Function `requireTenant(req, res, next)`:
- Check `req.session?.user` exists, else 401
- Extract `tenantId` from `req.session.currentOrgId` (set during login)
- If no tenantId, return 403 `{ error: 'No organization selected' }`
- Attach `(req as any).tenantId = tenantId`
- Call next()

Function `getTenantId(req: Request): string | null`:
- Return `(req as any).tenantId || req.session?.currentOrgId || null`

Export both functions.

**Step 2: Add requireAbility middleware to auth.middleware.ts**

Add new middleware factory to `be/src/shared/middleware/auth.middleware.ts`:

```typescript
import { abilityService, AppAbility } from '@/shared/services/ability.service.js'
import { createMongoAbility } from '@casl/ability'
```

Add `requireAbility(action: string, subject: string)` middleware factory:
- Get user from session
- If no user, 401
- Try to load ability from Valkey via `abilityService.loadCachedAbility(req.sessionID)`
- If no cached ability, build fresh one: `abilityService.buildAbilityFor({ id: user.id, role: user.role, is_superuser: user.is_superuser, current_org_id: req.session.currentOrgId || '' })`
- Cache the fresh ability: `abilityService.cacheAbility(req.sessionID, ability)`
- Check `ability.can(action, subject)`, if false return 403 with `{ error: 'Access denied' }`
- Attach ability to request: `(req as any).ability = ability`
- Call next()

Keep ALL existing middleware functions unchanged (requireAuth, requireRole, requirePermission, requireOwnership, etc.) for backward compatibility.

**Step 3: Wire ability computation into login flow**

Read `be/src/modules/auth/auth.service.ts` to find the login function. After successful login (where `req.session.user` is set):
- Set `req.session.currentOrgId` to the user's default org (query user_tenant table to get first tenant_id for this user)
- Build ability: `const ability = abilityService.buildAbilityFor({ id: user.id, role: userTenantRole, is_superuser: user.is_superuser, current_org_id: tenantId })`
- Cache: `await abilityService.cacheAbility(req.sessionID, ability)`

The login flow should:
1. Authenticate user (existing logic)
2. Query user_tenant to get the user's org memberships
3. Set session.currentOrgId to first org (or SYSTEM_TENANT_ID as default)
4. Set session.user.role to the role from user_tenant for that org
5. Build and cache CASL ability

If user has no user_tenant entry, create one with role='user' and tenant_id=SYSTEM_TENANT_ID (backward compatibility for existing users).
