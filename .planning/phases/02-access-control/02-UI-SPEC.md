## Interaction Patterns

### 1. Role Assignment (RoleManagementTable)

**Location:** Data Studio > Users page (admin+ only)
**Trigger:** Admin navigates to user management
**Flow:**
1. Table displays all org members with columns: Name, Email, Current Role, Actions
2. Role column shows a `Select` dropdown pre-filled with current role
3. Changing the select triggers a confirmation dialog: "Change [user]'s role from [old] to [new]?"
4. On confirm, mutation fires, success toast shows, audit log entry created
5. If the role change affects the current user's session, redirect to login

**States:**
- Loading: Skeleton rows (3 rows, matching table column widths)
- Empty: EmptyState with `Users` icon, "No members in this organization"
- Error: Toast notification with retry action
- Saving: Select disabled, spinner in actions column

### 2. ABAC Policy Rule Editor (PolicyRuleEditor)

**Location:** Dataset create/edit form > "Access Policy" section
**Trigger:** Leader or admin edits a dataset's settings
**Flow:**
1. Section header: "Access Policy" with info tooltip explaining inheritance
2. Public/Private toggle (existing pattern from DatasetAccessDialog)
3. When private, show policy rules list:
   - Each rule row: [Effect: allow/deny] [Action: read/update/delete] [Conditions: key=value pairs]
   - "Add Rule" button appends a new empty rule row
   - Each row has a delete (X) button
4. Conditions are key-value pairs: attribute name (text input) + operator (select: equals, in, not equals) + value (text input)
5. Save applies to dataset's `policy_rules` JSONB column

**States:**
- No rules: Muted text "No access policy rules. All org members with appropriate roles can access this dataset."
- Adding rule: New row appears with empty fields, focus on effect select
- Validation error: Red border on invalid fields, error text below
- Saving: Button shows spinner, all inputs disabled

### 3. Conditional UI Rendering (AbilityProvider + Can)

**Pattern:** Wrap protected UI elements with `<Can>` from @casl/react
**Implementation:**
```
<Can I="create" a="Dataset">
  <Button>Create Dataset</Button>
</Can>
```

**Visibility rules:**
| Element | Visible To |
|---------|------------|
| Data Studio sidebar section | leader, admin, super-admin |
| Create Dataset button | leader, admin, super-admin |
| Delete Dataset button | admin, super-admin (or dataset creator if leader) |
| User Management page | admin, super-admin |
| Audit Log page | admin, super-admin |
| Org Settings page | admin, super-admin |
| Platform Settings | super-admin only |

**Behavior for unauthorized access via direct URL:** Show `AccessDenied` page (existing pattern at `fe/src/features/errors/`) with message: "You do not have permission to access this page. Contact your administrator for access."

### 4. Org Switcher (OrgSwitcher)

**Location:** Header bar, right side (next to user avatar)
**Trigger:** User belongs to multiple orgs
**Flow:**
1. Select dropdown shows current org name
2. Dropdown lists all orgs the user belongs to, with their role in parentheses
3. Selecting a different org triggers session update, page reload
4. Single-org users see org name as static text (no dropdown)

**States:**
- Single org: Static text display, no interactive element
- Multi-org: Select with org names + role labels
- Switching: Spinner overlay, all interaction disabled during transition

### 5. Extended Audit Log

**Changes to existing AuditLogPage:**
- New audit action badges added to `getActionBadge()` map (following existing color convention)
- New resource types added to `formatResourceType()` map
- Super-admin sees all-org view with org column and org filter dropdown
- Admin sees only their org's logs (current behavior, no change)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Role badges | Use `role="status"` and `aria-label="Role: admin"` |
| Policy rule delete | Confirm dialog before destructive action (not just icon button) |
| Org switcher | `aria-label="Switch organization"`, keyboard navigable select |
| Conditional rendering | Hidden elements use `<Can>` (not rendered at all, not `display:none`) -- no ARIA hiding needed |
| Audit log table | Existing table patterns with proper `<th>` scope already in place |
| Error announcements | Toast notifications use Sonner (already has `role="alert"`) |
