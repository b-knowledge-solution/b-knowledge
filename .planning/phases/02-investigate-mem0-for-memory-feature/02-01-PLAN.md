  <action>
Create `benchmarks/test_dealbreakers.py` with pytest test functions that verify each deal-breaker against LIVE infrastructure. The script must produce clear PASS/FAIL output for each deal-breaker.

**Test 1: `test_opensearch_connection`** (Deal-breaker #1 -- D-10)
- Use `mem0_config` fixture
- Create a `Memory.from_config(config)` instance
- Call `m.add(messages=[{"role": "user", "content": "Test message for OpenSearch"}, {"role": "assistant", "content": "Acknowledged"}], user_id="test_db1_user")` with `infer=False` to skip LLM extraction
- Call `m.search(query="test message", user_id="test_db1_user", limit=5)`
- Assert results list is non-empty
- Cleanup: delete the test memory and opensearch index via `cleanup_opensearch_index`

**Test 2: `test_multi_tenant_isolation`** (Deal-breaker #2 -- D-07)
- Create TWO mem0 configs with different `collection_name` values: `"mem0_test_tenant_A"` and `"mem0_test_tenant_B"`
- Create two Memory instances
- Add a memory to tenant A: `"Tenant A secret data"` with `user_id="tenantA:user1"`
- Add a memory to tenant B: `"Tenant B secret data"` with `user_id="tenantB:user1"`
- Search tenant A for `"secret data"` with `user_id="tenantA:user1"` -- assert results contain "Tenant A" NOT "Tenant B"
- Search tenant B for `"secret data"` with `user_id="tenantB:user1"` -- assert results contain "Tenant B" NOT "Tenant A"
- Cleanup both test indices

**Test 3: `test_custom_llm_provider`** (Deal-breaker #3 -- D-09, D-16)
- Create a mem0 config with explicit `llm.provider` set to `"openai"` and a custom `llm.config.model` value
- Create a Memory instance
- Call `m.add(messages=[{"role": "user", "content": "I prefer dark mode and TypeScript"}, {"role": "assistant", "content": "Noted your preferences."}], user_id="test_db3_user")` with `infer=True` (LLM extraction enabled)
- Assert the result contains extracted memories (non-empty `results` list with `event` field)
- This proves mem0 can use a configurable LLM provider for extraction
- Cleanup test index

**Test 4: `test_custom_embedding_provider`** (Deal-breaker #3 continued)
- Create a mem0 config with explicit `embedder.provider` and `embedder.config.model`
- Add a memory and then search -- search returning results proves embeddings were generated with the configured provider
- Cleanup test index

**Test 5: `test_licensing_verification`** (Deal-breaker #4)
- Run `.venv/bin/pip show mem0ai` and assert the license field contains "Apache" or check the LICENSE file in the installed package
- This is a static check, no infrastructure needed

**Test 6: `test_apache_age_pg17_compatibility`** (D-11)
- Use `check_age_extension` from `mem0_setup.py`
- If AGE extension is available: create a test graph, insert a node, query it, drop the graph. Assert all operations succeed.
- If AGE is NOT available (extension not installed in PG17 Docker image): mark test as `pytest.mark.skip` with message "Apache AGE not installed in PostgreSQL 17 -- manual installation required"
- Record the result either way -- this informs the ADR

**Test 7: `test_mem0_rest_api_server`** (D-12)
- Import `mem0.server` or the equivalent server module
- Start the server programmatically (or document the command: `mem0 server --port 8888`)
- Use `httpx` to call `GET /health` or `GET /config` on the server
- Assert the server responds with 200
- If the server cannot start without specific config, document the required config in a comment and use a simulated/mock approach
- Note: This test may need to be skipped if the server requires Neo4j/pgvector by default. Document the finding.

Each test function should print a clear result line: `DEAL-BREAKER #{N}: PASS` or `DEAL-BREAKER #{N}: FAIL -- {reason}`.

At the end of the file, add a `test_summary` function that collects all results and prints a summary table matching the format in 02-RESEARCH.md deal-breaker table.
