       ```typescript
       interface FeedbackCommentPopoverProps {
         /** Current feedback state */
         feedback: 'up' | 'down' | null
         /** Called when user submits feedback. thumbup=true for up, false for down. comment is optional text. */
         onFeedback: (thumbup: boolean, comment?: string) => void
         /** Button size - sm for inline, md for standalone */
         size?: 'sm' | 'md'
         /** Whether buttons should be disabled (e.g. during streaming) */
         disabled?: boolean
       }
       ```

       Component behavior:
       - Renders ThumbsUp and ThumbsDown buttons (ghost variant, h-6 w-6 with h-3 w-3 icons per UI-SPEC)
       - ThumbsUp: one-click, calls `onFeedback(true)`, sets feedback to 'up', icon turns `text-green-500`
       - ThumbsDown: opens Popover (side="bottom", align="end"), sets feedback to 'down', icon turns `text-red-500`
       - Popover is 280px wide with 16px padding
       - Contains: heading "What was wrong?" (12px, text-muted-foreground), textarea (3 rows, max 500 chars, placeholder "Tell us what could be better..."), button row with "Skip for now" (ghost, 12px) and "Send feedback" (primary, 14px, disabled when textarea empty)
       - "Send feedback" calls `onFeedback(false, comment.trim())`, closes popover, resets comment
       - "Skip for now" calls `onFeedback(false)`, closes popover, resets comment
       - Clicking outside popover: close without submitting, revert thumb state to null
       - Re-clicking active thumb toggles it off (feedback becomes null, no API call)
       - `aria-pressed` on both thumb buttons reflecting state
       - `disabled` prop applies `opacity-50 cursor-not-allowed` to both buttons
