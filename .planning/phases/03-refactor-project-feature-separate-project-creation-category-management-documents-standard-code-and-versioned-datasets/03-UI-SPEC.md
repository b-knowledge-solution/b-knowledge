## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Compact element spacing, tab content gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding within tabs, sidebar internal padding |
| xl | 32px | Tab content area padding, page header padding |
| 2xl | 48px | Major section breaks between page zones |
| 3xl | 64px | Not used in this phase |

Exceptions: 44px minimum touch target for category sidebar items on mobile viewports.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Display is used only for project name on the detail page header.
- Heading is used for tab section titles and category names.
- Body/Label share 14px (established codebase pattern from existing project pages).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` / white light, slate-900 dark | Page background, tab content areas |
| Secondary (30%) | `hsl(var(--card))` / white light, slate-800 dark | Project cards, category sidebar, version list cards |
| Accent (10%) | `#0D26CF` / `var(--primary-color)` | See reserved list below |
| Destructive | `hsl(var(--destructive))` / red-500 light, red-700 dark | Delete project, delete category, delete version buttons only |

Accent reserved for:
- "Create Project" primary CTA button
- "New Category" primary CTA button
- "New Version" primary CTA button
- Active tab indicator (Documents / Standard / Code)
- Active category item highlight in sidebar
- Version badge for latest/current version

## Category Type Visual Differentiation

| Type | Icon | Badge Color (light / dark) |

| Code | `Code2` | `bg-emerald-100 text-emerald-700` / `bg-emerald-900/30 text-emerald-300` |

## Page Layout Contracts

### Project List Page

| Element | Specification |
|---------|--------------|
| Layout | Header row (title + create button) + responsive card grid |
| Grid | `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4` |
| Card content | Project name (heading), description (body, 2-line clamp), visibility badge (public/private), category count summary |
| Card interaction | Entire card is clickable, navigates to detail page. No edit/delete on card. |
| Card hover | `hover:border-primary/50` border highlight, no elevation change |
| Empty state | Shown when zero projects exist |
| Loading state | Skeleton cards (3 per row) using existing `skeleton.tsx` |

### Project Detail Page

| Element | Specification |
|---------|--------------|
| Layout | Back button + Header (project name display, status badge) + 3 category tabs + settings sidebar |
| Tab bar | `Tabs` component with 3 fixed tabs: Documents, Standard, Code. Each tab label includes icon + text + count badge. |
| Tab content | Category list (left sidebar, 240px fixed width) + content area (remaining width) |
| Category sidebar | Scrollable list of categories within the active tab. Active item highlighted with `bg-accent` + left border accent. "New Category" button pinned at bottom. |
| Content area | Changes based on category type — see below |
| Settings | Collapsible sidebar on the right (320px), toggled via gear icon button in header. Contains: edit project name/description, visibility toggle, member management, danger zone (delete). |

### Documents Category Content

| Element | Specification |
|---------|--------------|
| Version list | Vertical list of version cards, newest first. Each card: version label (e.g., "v3"), creation date, file count, parsing status badge. |
| Active version | Click to expand inline — shows file list via existing `DocumentListPanel`. |
| New Version button | Secondary button at top of version list. Opens `VersionModal`. |
| Version status badges | `parsing` = amber spinner, `ready` = green check, `error` = red x, `archived` = muted/gray |

### Standard Category Content

| Element | Specification |
|---------|--------------|
| Layout | Single dataset view — no version list. Directly shows `DocumentListPanel` with the auto-created dataset. |
| Header | Category name + parser config summary + "Upload Files" button |
| Parser override | Gear icon button opens parser config popover/sheet |

### Code Category Content

| Element | Specification |
|---------|--------------|
| Layout | Single dataset view (same as Standard) + optional git sync panel |
| Header | Category name + detected languages badge list + "Upload Files" button |
| Git sync panel | Collapsible section below header. Fields: repository URL, branch, auth token (masked), sync interval dropdown. Status indicator (last sync time, next sync). Manual "Sync Now" button. |
| Language badges | Auto-detected from uploaded files. Use `Badge` variant="outline" with `Code2` icon. |

## Interaction Contracts

### Navigation

| From | Action | To |
|------|--------|----|
| Project list | Click card | Project detail page (`/projects/:id`) |
| Project detail | Click back button | Project list page (`/projects`) |
| Project detail | Click tab | Switch category type view (URL param: `?tab=documents`) |
| Category sidebar | Click category | Select category, show its content |
| Version list | Click version card | Expand version inline, show file list |

### State Persistence

| State | Mechanism | Reason |
|-------|-----------|--------|
| Active tab | URL search param `?tab=documents` | Bookmarkable, survives refresh |
| Active category | URL search param `?category=<id>` | Bookmarkable, deep-linkable |
| Settings sidebar open/closed | Local `useState` | Transient UI state |

### Loading States

| View | Loading behavior |
|------|-----------------|
| Project list | 3x4 skeleton card grid |
| Project detail header | Skeleton text for name + badge |
| Category sidebar | 4 skeleton list items (h-10) |
| Version list | 3 skeleton cards (h-16) |
| File list | Existing DocumentListPanel skeleton |

### Error States

| View | Error behavior |
|------|---------------|
| Project list | Full-page error with retry button |
| Project detail (404) | "Project not found" with back-to-list link |
| Category load failure | Inline error in content area with retry |
| Version load failure | Inline error in version list with retry |
| Delete failure | Toast notification via globalMessage |
