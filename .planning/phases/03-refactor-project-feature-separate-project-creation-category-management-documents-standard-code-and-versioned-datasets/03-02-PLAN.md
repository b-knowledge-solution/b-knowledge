**Implementation -- modify `createCategory` method in `ProjectCategoryService`:**

1. Accept `category_type` from data (default to 'documents')
2. Include `category_type` in the create payload
3. After category creation, if type is 'standard' or 'code':
   - Look up the project via `ModelFactory.project.findById(projectId)` to get default_embedding_model, default_chunk_method
   - Determine parser_id: if type='code' use 'code', else use project.default_chunk_method || 'naive'
   - Create dataset via `ModelFactory.dataset.create({ name: '${project.name}_${data.name}', parser_id, embedding_model: project.default_embedding_model, ... })`
   - Link dataset via `ModelFactory.projectDataset.create({ project_id: projectId, dataset_id: dataset.id, auto_created: true })`
   - Update category with dataset_id via `ModelFactory.documentCategory.update(category.id, { dataset_id: dataset.id })`
4. If type is 'documents', no dataset creation (existing behavior)

**Implementation -- modify `deleteCategory` method:**

1. Before deleting, look up the category via `ModelFactory.documentCategory.findById(categoryId)`
2. If category has `dataset_id` (standard/code type), soft-delete the dataset: `ModelFactory.dataset.update(category.dataset_id, { status: 'inactive' })`
3. Then delete the category (existing behavior)

Key implementation details:
- Dataset name format: `${project.name}_${category.name}` for standard/code (per RESEARCH.md)
- parser_id for code: literal string `'code'` (maps to advance-rag/rag/app/code.py parser)
- parser_id for standard: `project.default_chunk_method || 'naive'`
- Wrap dataset creation in try/catch (non-blocking, log warning on failure -- same pattern as createVersion)
- Use `log` from `@/shared/services/logger.service.js` for warnings
