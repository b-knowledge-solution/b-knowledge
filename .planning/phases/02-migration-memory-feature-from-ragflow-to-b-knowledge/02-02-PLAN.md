    1. Create `memory.service.ts` (singleton pattern like agentService):
       - `class MemoryService` with singleton export `memoryService`
       - `createPool(dto: CreateMemoryDto, userId: string, tenantId: string): Promise<Memory>`:
         Creates via `ModelFactory.memory.create({...dto, tenant_id: tenantId, created_by: userId})`.
         After creation, calls `memoryMessageService.ensureIndex(tenantId)`.
       - `listPools(tenantId: string, userId: string): Promise<Memory[]>`:
         Returns pools where `tenant_id = tenantId` AND (`permission = 'me' AND created_by = userId`) OR `permission = 'team'`. Order by created_at DESC.
       - `getPool(id: string, tenantId: string): Promise<Memory | null>`:
         `ModelFactory.memory.findById(id)` with tenant_id check.
       - `updatePool(id: string, dto: UpdateMemoryDto, tenantId: string): Promise<Memory>`:
         `ModelFactory.memory.update(id, dto)` with tenant check. Return updated.
       - `deletePool(id: string, tenantId: string): Promise<void>`:
         Delete all messages from OpenSearch (`memoryMessageService.deleteAllByMemory(id, tenantId)`), then delete pool record.

    2. Create `memory-message.service.ts` (singleton pattern):
       - `class MemoryMessageService` with singleton export `memoryMessageService`
       - Private `indexCache = new Set<string>()` for caching ensureIndex results per tenant (Pitfall 6)
       - `ensureIndex(tenantId: string): Promise<void>`:
         Index name = `memory_${tenantId}`. Check `indexCache`, if present skip. Otherwise `client.indices.exists()`, if not exists `client.indices.create()` with mapping from Research Pattern 3 (knn_vector dimension 1024, text field for content, keyword fields for IDs, date fields for timestamps). Add tenantId to indexCache.
       - `insertMessage(tenantId: string, doc: Omit<MemoryMessageDoc, 'created_at'>): Promise<void>`:
         Set `created_at = new Date().toISOString()`. Use `client.index()` with `index: memory_${tenantId}`, `id: doc.message_id`, `body: doc`. Then call `enforeFifo(tenantId, doc.memory_id)`.
       - `enforceFifo(tenantId: string, memoryId: string): Promise<void>`:
         Get pool from DB for memory_size. Query OpenSearch count for this memory_id. If count * 9000 (approx 9KB per message) > memory_size, calculate excess = count - Math.floor(memory_size / 9000). Delete oldest N messages via `deleteByQuery` with sort by created_at ASC, size = excess (D-07).
       - `searchMemory(tenantId: string, memoryId: string, query: string, queryVector: number[], topK = 10, vectorWeight = 0.7): Promise<MemorySearchResult[]>`:
         Hybrid search following Research Pattern 5. Bool query with must:[term memory_id], filter:[term tenant_id, term status:1, optional time range on valid_at/invalid_at], should:[match content with boost 1-vectorWeight, knn content_embed with k=topK]. Return mapped hits with score.
       - `listMessages(tenantId: string, memoryId: string, page: number, pageSize: number, keyword?: string, messageType?: number): Promise<{items: MemoryMessageDoc[], total: number}>`:
         OpenSearch search with bool filter on memory_id + tenant_id + optional keyword match + optional message_type filter. Sort by created_at DESC. Paginate with from/size.
       - `deleteMessage(tenantId: string, messageId: string): Promise<void>`:
         `client.delete({ index: memory_${tenantId}, id: messageId })`.
       - `deleteAllByMemory(memoryId: string, tenantId: string): Promise<void>`:
         `client.deleteByQuery({ index: memory_${tenantId}, body: { query: { term: { memory_id: memoryId } } } })`.
       - `updateMessageStatus(tenantId: string, messageId: string, status: number): Promise<void>`:
         `client.update({ index: memory_${tenantId}, id: messageId, body: { doc: { status } } })`.

       Use getClient() from opensearch-conn following existing rag-search.service.ts pattern. tenantId is always first parameter per project decision.
