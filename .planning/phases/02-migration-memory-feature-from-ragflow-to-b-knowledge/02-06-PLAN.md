    4. Add memory_read and memory_write handlers to `node_executor.py`:

       - `async def handle_memory_read(node_config: dict, input_data: dict, context: dict) -> dict`:
         ```python
         """Read relevant memories for the given query.

         Args:
             node_config: Node configuration with memory_id, top_k, vector_weight.
             input_data: Input containing 'query' text from upstream node.
             context: Execution context with tenant_id, api_base_url, auth_token.

         Returns:
             Dict with 'memories' key containing list of relevant memory content strings.
         """
         ```
         a. Extract memory_id from node_config
         b. Extract query from input_data (use 'query' or 'input' or 'message' field)
         c. HTTP POST to `{api_base_url}/api/memory/{memory_id}/search` with auth header
            body: { query, top_k: node_config.get('top_k', 5) }
         d. Return { memories: [item['content'] for item in response], raw_results: response }

       - `async def handle_memory_write(node_config: dict, input_data: dict, context: dict) -> dict`:
         ```python
         """Write content to a memory pool.

         Args:
             node_config: Node configuration with memory_id.
             input_data: Input containing 'content' text to store as memory.
             context: Execution context with tenant_id, user_id, session_id.

         Returns:
             Dict with 'status' key indicating success.
         """
         ```
         a. Extract memory_id, content from config and input
         b. HTTP POST to `{api_base_url}/api/memory/{memory_id}/messages` with auth header
            body: { content, message_type: node_config.get('message_type', 1) }
         c. Return { status: 'stored', memory_id }

       - Register both in HANDLERS dict:
         `'memory_read': handle_memory_read, 'memory_write': handle_memory_write`
