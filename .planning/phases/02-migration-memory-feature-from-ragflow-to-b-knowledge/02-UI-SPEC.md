## Visual Focal Points

**Memory List Page (`/memory`):** The focal point is the card grid. The eye enters at the top-left card and scans right then down. The "Create Memory Pool" primary CTA in the top-right corner is the secondary focal point. Each card surfaces: pool name, memory type badges, scope badge, message count, and last updated timestamp.

**Memory Detail Page (`/memory/:id`):** Two-tab layout (Messages / Settings). The Messages tab focal point is the message table with search bar above. The Settings tab focal point is the form fields, with the "Save Settings" button anchored at the bottom of the form area.

**Accessibility note:** All memory type badge colors are paired with text labels (never color-only semantics). Table rows are keyboard-navigable with visible focus rings.

## Interaction Patterns

### Memory Pool CRUD

| Interaction | Behavior |
|-------------|----------|
| Create pool | Click "Create Memory Pool" > Dialog with form > Submit creates pool and navigates to detail page |
| Edit pool | Pool card kebab > "Edit" > Dialog with pre-filled form |
| Delete pool | Pool card kebab > "Delete" > Confirmation dialog (type pool name) |
| View detail | Click pool card > Navigates to `/memory/:id` |

### Message Browser

| Interaction | Behavior |
|-------------|----------|
| Search | Text input with 300ms debounce. Searches message content via hybrid vector+text search. |
| Filter by type | Toggle buttons for Raw/Semantic/Episodic/Procedural. Multiple selection allowed. Filters client-side from loaded results. |
| Delete message | Row kebab > "Delete" > Simple confirmation dialog (no name typing). |
| Clear all | Table toolbar "Clear All" > Confirmation dialog (type pool name). |
| Pagination | Cursor-based pagination with "Load more" button at table bottom. 50 messages per page. |
| View message | Click row to expand inline (accordion-style) showing full content, metadata, timestamps. |

### Settings Panel

| Interaction | Behavior |
|-------------|----------|
| Save | Click "Save Settings" at form bottom. Toast: "Settings saved". |
| Embedding model | Select dropdown populated from tenant LLM providers (embedding type). |
| LLM model | Select dropdown populated from tenant LLM providers (chat type). |
| Custom prompts | Textarea fields for system_prompt and user_prompt. Placeholder shows default prompt text. |
| Memory size | Slider (100 to 10,000, step 100). Label shows current value. Helper text explains FIFO behavior. |
| Extraction mode | Switch toggle: "Real-time extraction" with helper text "Extract memories after each conversation turn (in addition to session-end batch)." |

### Import History

| Interaction | Behavior |
|-------------|----------|
| Trigger | Pool card kebab > "Import Chat History" OR detail page toolbar button "Import History" |
| Dialog | Shows list of chat conversations (name, message count, date). Checkbox multi-select. |
| Execute | Click "Import Selected" > Dialog shows progress bar > Toast on completion. |
| Progress | Progress bar with "{processed}/{total} conversations" label below. Dialog cannot be dismissed during import. |

### Agent Canvas Memory Node

| Interaction | Behavior |
|-------------|----------|
| Add node | Drag Memory from palette or Cmd+K > "Memory". |
| Configure | Double-click node > Config panel shows MemoryForm. |
| Form fields | Pool selector (dropdown of user's pools), mode toggle (Read/Write), query input (for read mode). |

## Layout Contract

### Memory List Page (`/memory`)

```
+--sidebar--+--main-content-area-------------------+
|            | [heading] Memory Pools    [+ Create] |
|  Agents    | [search bar]           [scope filter] |
|  Memory *  |                                      |
|  Chat      | [card] [card] [card]                 |
|  Search    | [card] [card] [card]                 |
|  ...       | [card] [card] [card]                 |
+------------+--------------------------------------+
```

- Card grid: 3 columns on desktop (>1280px), 2 columns (>768px), 1 column (mobile)
- Card content: pool name (14px/600), description truncated to 2 lines, memory type badges (colored), scope badge, message count, last updated
- Card actions: kebab menu (Edit, Import Chat History, Clear All, Delete)
- Sidebar: Memory appears as a child under Agents top-level item (per D-12), using `Brain` icon from lucide-react

### Memory Detail Page (`/memory/:id`)

```
+--sidebar--+--main-content-area-----------------------------------+
|            | [< Back] [Pool Name]              [Import] [More v] |
|            | [tabs] Messages | Settings                          |
|            +-----------------------------------------------------+
|            | Messages tab:                                        |
|            | [search bar] [type filter: Raw Sem Epi Pro]          |
|            | +--table-----------------------------------------+   |
|            | | Type | Content (truncated) | Created | Actions |   |
|            | | [badge] | First line of...  | 2h ago  | [...]   |   |
|            | | [badge] | Another memory... | 1d ago  | [...]   |   |
|            | +------------------------------------------------+   |
|            | [Load more]                                          |
|            +-----------------------------------------------------+
|            | Settings tab:                                        |
|            | [form max-width: 640px, centered]                    |
|            | Embedding Model: [select]                            |
|            | LLM Model: [select]                                  |
|            | Memory Size: [slider] 1000                           |
|            | Extraction Mode: [switch] Real-time                  |
|            | System Prompt: [textarea]                            |
|            | User Prompt: [textarea]                              |
|            | [Save Settings]                                      |
+------------+-----------------------------------------------------+
```

- Detail page occupies full main content area (no side panels)
- Tabs are flush with the content area top (below page header row)
- Message table uses full available width
- Settings form has max-width 640px, left-aligned within content area
- Page header: Back button (left), pool name as Heading (20px/600), Import button + kebab menu (right)

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Table keyboard navigation | Tab through table rows. Enter to expand message detail. Arrow keys to move between rows. |
| Screen reader memory types | Badge text includes full type name ("Semantic", not just color). aria-label on MemoryTypeBadges: "Memory types: Semantic, Episodic" |
| Focus visible | All interactive elements use `ring` token for focus outline |
| Color contrast | Memory type badge colors used with text labels (not color-only semantics) |
| Reduced motion | Import progress bar respects `prefers-reduced-motion` (disable animation, show percentage text) |
| Touch targets | All table row actions and filter toggles minimum 44x44px |
| Form labels | Every form field in settings panel has explicit `<Label>` association via `htmlFor` |
| Dialog focus trap | Create/Edit/Import/Delete dialogs trap focus within dialog while open |
