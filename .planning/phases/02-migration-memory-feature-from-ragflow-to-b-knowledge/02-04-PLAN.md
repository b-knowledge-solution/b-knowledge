  <action>
    1. Create `memory-extraction.service.ts` (singleton):
       - `class MemoryExtractionService` with export `memoryExtractionService`

       - `extractFromConversation(memoryId: string, userInput: string, assistantResponse: string, sessionId: string, userId: string, tenantId: string): Promise<void>`:
         a. Load memory pool from DB via `ModelFactory.memory.findById(memoryId)`
         b. If not found, log warning and return
         c. Combine conversation text: `User: ${userInput}\nAssistant: ${assistantResponse}`
         d. Determine which memory types to extract based on `memory.memory_type` bitmask:
            - If bitmask & 1 (RAW): store raw conversation text directly (no LLM call)
            - If bitmask & 2 (SEMANTIC): call LLM with semantic prompt
            - If bitmask & 4 (EPISODIC): call LLM with episodic prompt
            - If bitmask & 8 (PROCEDURAL): call LLM with procedural prompt
         e. For each enabled type, call `extractByType()`
         f. Generate embedding for each extracted item via `getEmbedding()`
         g. Insert each into OpenSearch via `memoryMessageService.insertMessage()`

       - Private `extractByType(conversation: string, type: number, memory: Memory): Promise<string[]>`:
         a. Select prompt template based on type (RAW/SEMANTIC/EPISODIC/PROCEDURAL)
         b. If memory has custom system_prompt/user_prompt (D-03), use those instead of defaults
         c. Replace `{{conversation}}` placeholder in user prompt with actual conversation text
         d. Call `llmClientService.chat()` with the memory's `llm_id` model (or tenant default if null)
         e. Parse LLM response as JSON array of strings (Pitfall 3: wrap in try/catch, use regex extraction fallback `\[[\s\S]*?\]` to find JSON array in response, fall back to [response] if parsing fails)
         f. Return extracted strings array

       - Private `getEmbedding(text: string, embdId: string | null, tenantId: string): Promise<number[]>`:
         a. Call llmClientService.embed() with the memory's embd_id model (or tenant default if null)
         b. Return embedding vector

       - `extractBatch(memoryId: string, conversationHistory: Array<{role: string, content: string}>, sessionId: string, userId: string, tenantId: string): Promise<void>`:
         For batch mode (D-02): concatenate full conversation history into single text, then call extractFromConversation with combined text.

       - `importChatHistory(memoryId: string, sessionId: string, userId: string, tenantId: string): Promise<{imported: number}>`:
         For D-11: Load all messages from chat session via ModelFactory.chatMessage.
         Group into conversation pairs (user + assistant). Process each pair through extractFromConversation.
         Return count of imported memory items.

    2. Update barrel index.ts to export memoryExtractionService
  </action>
