## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, toggle button gaps (gap-1.5 = 6px rounds to 8px in context) |
| md | 16px | Default element spacing, dialog body padding |
| lg | 24px | Section padding, dialog header/footer padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Toggle button row uses `gap-1.5` (6px) matching existing ChatInput pattern. Touch target for trigger button is 28x28px (`h-7 w-7`), matching existing toggle buttons in ChatInput.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` / white | Dialog background, textarea background |
| Secondary (30%) | `hsl(var(--muted))` / slate-100 | Step indicator inactive circles, keyword search input bg, inactive toggle buttons |
| Accent (10%) | `#0D26CF` / `hsl(var(--primary))` | Active step indicator circles, "Apply to Chat" button, step divider lines for completed steps |
| Destructive | `hsl(var(--destructive))` / red-500 | Not used in this phase (no destructive actions) |

Accent reserved for:
- Step indicator active/completed circles (`bg-primary text-primary-foreground`)
- "Apply to Chat" primary action button
- Step divider lines for completed steps (`bg-primary`)
- Sparkles trigger button hover state (inherits existing toggle button pattern: `text-slate-400` default, `hover:text-slate-600` on hover)

Note: The Sparkles trigger button does NOT have an active/highlighted state (it is a one-shot action, not a toggle). It uses the same idle styling as the file upload button: `text-slate-400 dark:text-slate-500 hover:text-slate-600 dark:hover:text-slate-300 hover:bg-slate-100 dark:hover:bg-slate-700`.

## Interaction Contract

### Trigger: Sparkles Button

| Property | Value |
|----------|-------|
| Location | ChatInput toggle row, leftSlot, after AssistantSelectorPopover |
| Size | 28x28px (`h-7 w-7 rounded-lg`) |
| Icon | `Sparkles` at 16x16px (`h-4 w-4`) |
| Idle state | `text-slate-400 dark:text-slate-500` |
| Hover state | `hover:text-slate-600 dark:hover:text-slate-300 hover:bg-slate-100 dark:hover:bg-slate-700` |
| Active state | None (one-shot action, not a toggle) |
| Tooltip | `title={t('glossary.promptBuilder.title')}` (native title attribute, matching existing buttons) |
| Click action | Opens PromptBuilderModal |

### Modal: PromptBuilderModal

| Property | Value |
|----------|-------|
| Trigger | Sparkles button click |
| Size | Default shadcn Dialog width (max-w-lg or as existing) |
| Close | X button, backdrop click (existing behavior) |
| Steps | 3: Language, Task, Keywords + Context |
| Apply action | Calls `onApply(prompt)` which sets ChatInput textarea value via imperative handle |
| Post-apply | Modal closes, textarea receives prompt text, textarea auto-resizes |

### Apply Flow

1. User clicks "Apply to Chat" in modal
2. Modal calls `onApply(promptText)`
3. ChatPage handler calls `chatInputRef.current?.setValue(promptText)`
4. `setValue` sets `textareaRef.current.value = text` and triggers auto-resize
5. Modal closes via `onClose()`
6. User can review/edit the prompt in textarea before sending
7. All other chat state (file attachments, reasoning toggle, internet toggle) is preserved (D-07)
