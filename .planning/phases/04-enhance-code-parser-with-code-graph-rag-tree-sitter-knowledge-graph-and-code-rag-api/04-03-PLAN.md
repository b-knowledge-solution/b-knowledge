# Plan 04-03: Task Executor Integration + Tests

## Objective
Wire `chunk_with_graph()` into the task executor pipeline so code files automatically get their knowledge graph extracted and stored in Memgraph. Add unit tests for the graph extraction module.

## must_haves
- Code parser in task executor calls `chunk_with_graph()` instead of `chunk()`
- Graph extraction failures don't break the chunk pipeline
- Unit tests for Python, TS, Java graph extraction
- Tests verify correct nodes and relationships

<action>
In `task_executor.py`, modify `build_chunks()` to detect code parser and use graph extraction:

At line ~430 where `cks = await thread_pool_exec(chunker.chunk, ...)`:

```python
# For code parser, also extract graph
if task["parser_id"].lower() == ParserType.CODE.value:
    async with chunk_limiter:
        cks, graph_ok = await thread_pool_exec(
            chunker.chunk_with_graph,
            task["name"],
            binary=binary,
            from_page=task["from_page"],
            to_page=task["to_page"],
            lang=task["language"],
            callback=progress_callback,
            kb_id=task["kb_id"],
            parser_config=task["parser_config"],
            tenant_id=task["tenant_id"],
        )
    if graph_ok:
        logging.info(f"Code graph extracted for {task['name']}")
    else:
        logging.warning(f"Code graph extraction skipped for {task['name']}")
else:
    async with chunk_limiter:
        cks = await thread_pool_exec(
            chunker.chunk,
            task["name"],
            binary=binary,
            ...existing args...
        )
```

This keeps backward compatibility — only code parser triggers graph extraction.
</action>
