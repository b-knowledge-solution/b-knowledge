# Plan 04-03: Task Executor Integration and Tests

## Objective

Wire the code graph extraction into the task executor pipeline so that code files automatically produce knowledge graph data alongside standard chunks. Add comprehensive tests for both the graph extraction module and the code-RAG query module.

## must_haves
- [ ] Task executor calls `chunk_with_graph()` for code parser and stores the graph
- [ ] Unit tests for code_graph.py covering Python, TypeScript, and Java
- [ ] Unit tests for code_rag_query.py covering search functions
- [ ] Existing code parser tests still pass (no regressions)

## Tasks

<task id="03-01" wave="2">
<title>Integrate code graph into task executor</title>

<read_first>
- advance-rag/rag/svr/task_executor.py (find where parser_id == "code" is handled, where chunk() is called)
- advance-rag/rag/graphrag/utils.py (set_graph function signature and usage)
- advance-rag/rag/app/code.py (chunk_with_graph function added in Plan 01)
</read_first>

<action>
Modify `advance-rag/rag/svr/task_executor.py`:

1. Find the section where `parser_id == "code"` triggers the code parser's `chunk()` function
2. Replace the `chunk()` call with `chunk_with_graph()` from `rag.app.code`
3. After getting `(chunks, graph)` back:
   - Process chunks as before (store in OpenSearch via existing pipeline)
   - If `graph` is not None and has nodes:
     - Import `set_graph`, `GraphChange` from `rag.graphrag.utils`
     - Create a `GraphChange` with all nodes as `added_updated_nodes` and all edges as `added_updated_edges`
     - Call `await set_graph(tenant_id, kb_id, embd_mdl, graph, change, callback)` to persist the code knowledge graph
     - Log: `f"Code graph stored: {graph.number_of_nodes()} nodes, {graph.number_of_edges()} edges"`
4. Fallback: if `chunk_with_graph()` fails, fall back to standard `chunk()` and log warning

Key import to add:
```python
from rag.app.code import chunk_with_graph
```

The existing code pipeline flow in task_executor.py handles embedding and indexing the chunks. The graph storage is an additional step after chunk indexing.
</action>

<acceptance_criteria>
- `advance-rag/rag/svr/task_executor.py` contains `chunk_with_graph` (grep: `chunk_with_graph`)
- File contains `set_graph` import or usage (grep: `set_graph`)
- File contains `GraphChange` import or usage (grep: `GraphChange`)
- File contains logging message about code graph (grep: `Code graph stored` or `code graph`)
</acceptance_criteria>
</task>
