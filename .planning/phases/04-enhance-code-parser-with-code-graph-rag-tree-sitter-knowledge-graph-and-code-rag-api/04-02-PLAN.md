# Plan 04-02: Code Graph Parser Pipeline

## Objective
Port the full code-graph-rag parser pipeline into b-knowledge's advance-rag, creating the `rag/app/code_graph/` module with 12-language support, and add a `chunk_with_graph()` pipeline interface to `code.py`.

## must_haves
- All 12 languages supported: Python, JS, TS, Rust, Java, C, C++, Lua, Go, Scala, C#, PHP
- ProcessorFactory wiring: Structure → Import → Definition → TypeInference → Call
- Deep cross-file call resolution via import mapping + function registry
- Memgraph Bolt client for storing extracted graph
- `chunk_with_graph()` in code.py returning (chunks, graph_data)
- Nested function/class hierarchy with FQN (fully qualified names)

   ```python
   class MemgraphIngestor:
       """Writes code graph nodes and relationships to Memgraph via Bolt."""
       
       def __init__(self, bolt_url: str, kb_id: str):
           self.driver = neo4j.GraphDatabase.driver(bolt_url)
           self.kb_id = kb_id
           self._node_batch = []
           self._rel_batch = []
       
       def ensure_node_batch(self, label, properties):
           properties['kb_id'] = self.kb_id
           self._node_batch.append((label, properties))
       
       def ensure_relationship_batch(self, from_id, rel_type, to_id):
           self._rel_batch.append((from_id, rel_type, to_id))
       
       def flush(self):
           """Write batched nodes and relationships to Memgraph."""
           # Use MERGE to avoid duplicates
           # Batch in transactions of 1000
       
       def close(self):
           self.driver.close()
   ```

```python
def chunk_with_graph(filename, binary, from_page=0, to_page=100000000,
                     lang="English", callback=None, kb_id="", 
                     parser_config=None, tenant_id=""):
    """Extended chunk function that also extracts code knowledge graph.
    
    Runs the standard chunk() for text chunks (→ OpenSearch), then runs
    the graph extraction pipeline (→ Memgraph) for code structure.
    
    Args:
        filename: Source file name.
        binary: Raw file bytes.
        kb_id: Knowledge base ID for graph partitioning.
        (other args same as chunk())
    
    Returns:
        tuple: (chunks: list[dict], graph_extracted: bool)
    """
    # 1. Run existing chunk() for text chunks
    chunks = chunk(filename, binary, from_page=from_page, to_page=to_page,
                   lang=lang, callback=callback, kb_id=kb_id,
                   parser_config=parser_config, tenant_id=tenant_id)
    
    # 2. Run graph extraction pipeline
    graph_extracted = False
    try:
        from rag.app.code_graph import extract_code_graph
        graph_extracted = extract_code_graph(
            filename=filename,
            binary=binary,
            kb_id=kb_id,
            callback=callback,
        )
    except Exception as e:
        logging.warning(f"Code graph extraction failed for {filename}: {e}")
    
    return chunks, graph_extracted
```

Also create the public `extract_code_graph()` function in `rag/app/code_graph/__init__.py` that:

2. Creates MemgraphIngestor

5. Flushes and closes the ingestor
