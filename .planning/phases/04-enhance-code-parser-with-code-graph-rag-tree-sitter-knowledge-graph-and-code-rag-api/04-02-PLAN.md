# Plan 04-02: Code-RAG Query API

## Objective

Create a code-specific RAG query module that can search the code knowledge graph using the existing KGSearch infrastructure. This provides a specialized retrieval path for code structure queries (e.g., "What functions call calculate_discount?", "Show me the class hierarchy for OrderProcessor").

## must_haves
- [ ] New module `rag/app/code_rag_query.py` with code-specific graph search
- [ ] Search code entities by name, type, and relationships
- [ ] Format results with code snippets and relationship context
- [ ] Compatible with existing KGSearch retrieval patterns

## Tasks

<action>
Create `advance-rag/rag/app/code_rag_query.py`:

1. **CodeGraphSearch(KGSearch)** class extending existing KGSearch:
   - Override `query_rewrite()` to produce code-specific entity types: FILE, CLASS, FUNCTION, METHOD, MODULE
   - Add `search_code_entities(name, entity_type, filters, idxnms, kb_ids, emb_mdl)` — find entities by code name
   - Add `search_code_relations(entity_name, relation_type, filters, idxnms, kb_ids)` — find CALLS, IMPORTS, INHERITS, CONTAINS relationships for a given entity
   - Add `get_callers(function_name, ...)` — convenience: find all functions that CALL a given function
   - Add `get_callees(function_name, ...)` — convenience: find all functions CALLED by a given function
   - Add `get_class_hierarchy(class_name, ...)` — convenience: trace INHERITS chain

2. **CODE_QUERY_PROMPT** — specialized prompt for code-structure query rewriting:
   ```
   Given a user question about source code structure, extract:
   1. entity_names: specific function/class/file names mentioned
   2. relation_types: what relationships are being asked about (CALLS, IMPORTS, INHERITS, CONTAINS)
   3. entity_types: what types of code elements (FILE, CLASS, FUNCTION, METHOD)
   
   User question: {question}
   Output JSON: {"entity_names": [...], "relation_types": [...], "entity_types": [...]}
   ```

3. **code_retrieval(question, tenant_ids, kb_ids, emb_mdl, llm, ...)** method:
   - Parse question with CODE_QUERY_PROMPT to extract entity names and relation types
   - Search entities by name embeddings
   - Search relations by type and entity
   - Format results as structured code-context chunk with function signatures, call graph, and hierarchy
   - Return chunk dict compatible with existing RAG response format

All functions must have docstrings with @param/@returns, type hints, and use loguru for logging.
</action>
