# Plan 04-05: Frontend Graph Visualization

## Objective
Create an interactive code graph visualization page accessible from the project code category, with pan/zoom/click, node details, and PNG/SVG + JSON export.

## must_haves
- New feature module at `fe/src/features/code-graph/`
- Interactive graph at `/code-graph/:kbId`
- "Code Graph" button in project code category
- Click nodes to see source code snippets
- Export: PNG/SVG image + JSON graph data
- Dark mode support

## Tasks

<task id="05-01" title="Install graph visualization library and create API layer">
<read_first>
- fe/package.json (existing dependencies)
- fe/src/features/ (existing feature module patterns)
- fe/CLAUDE.md (API layer conventions: *Api.ts + *Queries.ts)
</read_first>
<action>
1. Install graph visualization library:
   ```bash
   cd fe && npm install react-force-graph-2d
   ```

2. Create `fe/src/features/code-graph/api/codeGraphApi.ts`:
   ```typescript
   import { apiClient } from '@/lib/apiClient'
   
   export const codeGraphApi = {
     getStats: (kbId: string) => 
       apiClient.get(`/code-graph/${kbId}/stats`),
     
     getFullGraph: (kbId: string) => 
       apiClient.get(`/code-graph/${kbId}/stats`), // reuse stats + graph
     
     queryGraph: (kbId: string, question: string) => 
       apiClient.post(`/code-graph/${kbId}/query`, { question }),
     
     getCallers: (kbId: string, name: string) => 
       apiClient.get(`/code-graph/${kbId}/callers/${name}`),
     
     getCallees: (kbId: string, name: string) => 
       apiClient.get(`/code-graph/${kbId}/callees/${name}`),
     
     getSnippet: (kbId: string, name: string) => 
       apiClient.get(`/code-graph/${kbId}/snippet/${name}`),
   }
   ```

3. Create `fe/src/features/code-graph/api/codeGraphQueries.ts` with TanStack Query hooks:
   - `useCodeGraphStats(kbId)`
   - `useCodeGraphFull(kbId)`
   - `useCodeGraphQuery(kbId)` mutation
   - `useCodeSnippet(kbId, name)`
</action>
<acceptance_criteria>
- `react-force-graph-2d` in fe/package.json
- codeGraphApi.ts exports API methods matching backend routes
- codeGraphQueries.ts exports TanStack Query hooks
- No `*Service.ts` naming (uses *Api.ts per convention)
</acceptance_criteria>
</task>

<task id="05-02" title="Create CodeGraphPage and GraphViewer component">
<read_first>
- fe/src/features/ (existing page patterns)
- fe/src/router.tsx (route registration)
- fe/CLAUDE.md (component conventions, dark mode)
</read_first>
<action>
1. Create `fe/src/features/code-graph/pages/CodeGraphPage.tsx`:
   - Full-page layout with graph viewer taking most space
   - Top bar: KB name, node/edge count stats, NL query input
   - Side panel (collapsible): node details, source code snippet
   - Bottom controls: zoom, filter by node type, export buttons

2. Create `fe/src/features/code-graph/components/GraphViewer.tsx`:
   - Uses `react-force-graph-2d` for the interactive graph
   - Color-coded nodes by type: Function=blue, Class=green, Module=gray, etc.

   - Hover → highlight connected nodes

3. Register route in `fe/src/router.tsx`:
   ```typescript
   { path: '/code-graph/:kbId', element: <CodeGraphPage /> }
   ```

4. Create `fe/src/features/code-graph/index.ts` barrel export

All i18n strings in 3 locales (en, vi, ja).
Dark mode support with class-based theming.
</action>
