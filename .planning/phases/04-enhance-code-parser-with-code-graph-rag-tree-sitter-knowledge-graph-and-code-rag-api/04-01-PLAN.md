# Plan 04-01: Code Graph Extraction Module

## Objective

Create `rag/app/code_graph.py` — a new module that extracts code structure relationships from Tree-sitter AST into a NetworkX graph. This module builds upon the existing code parser's AST traversal to produce a knowledge graph with nodes (File, Class, Function, Method) and edges (CONTAINS, CALLS, IMPORTS, INHERITS).

## must_haves
- [ ] Extract File, Class, Function/Method nodes from AST
- [ ] Extract CONTAINS relationships (File→Class, File→Function, Class→Method)
- [ ] Extract CALLS relationships by analyzing call_expression AST nodes
- [ ] Extract IMPORTS relationships from import statements
- [ ] Extract INHERITS relationships from class superclass lists
- [ ] Use qualified names for graph node identifiers (e.g., `module.ClassName.method_name`)
- [ ] Return a NetworkX graph compatible with existing `rag/graphrag/utils.py`
- [ ] Support Python, TypeScript, JavaScript, Java, Go, Rust (same languages as code parser)

<action>
Create `advance-rag/rag/app/code_graph.py` with:

1. **CALL_EXPRESSION_TYPES** dict mapping language → call expression AST node types:
   - python: ["call", "attribute"] 
   - javascript/typescript/tsx: ["call_expression", "new_expression"]
   - java: ["method_invocation", "object_creation_expression"]
   - go: ["call_expression"]
   - rust: ["call_expression", "macro_invocation"]
   - ruby: ["call", "method_call"]
   - c/cpp: ["call_expression"]
   - c_sharp: ["invocation_expression", "object_creation_expression"]

2. **SUPERCLASS_NODE_TYPES** dict mapping language → superclass AST node types:
   - python: ["argument_list"] (inside class_definition, the bases)
   - javascript/typescript: ["class_heritage"]
   - java: ["superclass", "super_interfaces"]
   - c_sharp: ["base_list"]
   - ruby: ["superclass"]

3. **extract_code_graph(filename, source_bytes, language, scope_nodes, imports_text) -> nx.Graph**:
   - Build graph with `nx.DiGraph()`
   - Create File node: `{entity_type: "FILE", description: f"Source file: {filename}", source_id: filename}`
   - For each scope_node in scope_nodes:
     - If class: create Class node with qualified_name = `module.ClassName`, add CONTAINS edge from File
     - If function/method: create Function/Method node with qualified_name, add CONTAINS edge from parent (File or Class)
   - Extract CALLS edges: walk function body AST nodes, find call_expression types, resolve callee name, add edge
   - Extract IMPORTS edges: parse import text to get module names, add IMPORTS edges from File node
   - Extract INHERITS edges: for each class node, find superclass names in AST, add INHERITS edge
   - Set graph.graph["source_id"] = [filename]
   - Return the NetworkX graph

4. **extract_call_targets(node, language) -> list[str]**: Recursively walk AST looking for call expressions, return list of called function/method names.

5. **extract_superclasses(node, language) -> list[str]**: Extract superclass names from class definition AST node.

6. **_resolve_qualified_name(name, scope_nodes, filename) -> str**: Resolve a simple name to fully qualified name using scope context.

All functions must have JSDoc-style docstrings with @param/@returns, type hints, and use loguru for logging.
</action>
