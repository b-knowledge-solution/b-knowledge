# Plan 04-01: Memgraph Infrastructure Setup

## Objective
Add Memgraph graph database to the b-knowledge infrastructure stack for storing code knowledge graphs with native Cypher support.

## must_haves
- Memgraph service running in Docker alongside existing infra
- Bolt port 7687 accessible from both Node.js backend and Python advance-rag
- Environment variables configured in all workspaces
- neo4j Python driver added to advance-rag dependencies

## Tasks

<task id="01-01" title="Add Memgraph service to docker-compose-base.yml">
<read_first>
- docker/docker-compose-base.yml (current infrastructure services)
- docker/.env (existing env vars pattern)
</read_first>
<action>
Add Memgraph service to `docker/docker-compose-base.yml`:

```yaml
memgraph:
  image: memgraph/memgraph:latest
  container_name: knowledge-memgraph
  ports:
    - "${MEMGRAPH_BOLT_PORT:-7687}:7687"
    - "${MEMGRAPH_WEB_PORT:-7444}:7444"
  volumes:
    - memgraph_data:/var/lib/memgraph
  command: ["--bolt-port=7687", "--log-level=WARNING", "--also-log-to-stderr"]
  restart: unless-stopped
  healthcheck:
    test: ["CMD-SHELL", "echo 'RETURN 1;' | mgconsole || exit 1"]
    interval: 10s
    timeout: 5s
    retries: 5
```

Add `memgraph_data` to the volumes section at the bottom of the file.
</action>
<acceptance_criteria>
- docker/docker-compose-base.yml contains `memgraph:` service block
- docker/docker-compose-base.yml contains `memgraph_data:` in volumes section
- Service uses image `memgraph/memgraph:latest`
- Bolt port maps to 7687
- healthcheck uses mgconsole
</acceptance_criteria>
</task>
