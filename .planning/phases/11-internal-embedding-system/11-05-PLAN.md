**1. Find and update the Dockerfile for advance-rag/task-executor:**

Search for the Dockerfile that builds the advance-rag image. It may be `docker/Dockerfile.task-executor`, `docker/Dockerfile.advance-rag`, or referenced in docker-compose.yml build context.

In that Dockerfile, add torch CPU-only installation BEFORE the `pip install` of pyproject.toml dependencies:

```dockerfile
# Install PyTorch CPU-only build FIRST (per D-03)
# Using separate index to avoid pulling CUDA wheels (~2.5GB vs ~250MB)
RUN pip install torch --index-url https://download.pytorch.org/whl/cpu --no-cache-dir
```

This must come before `pip install .` or `pip install -r requirements.txt` so that when sentence-transformers is installed via pyproject.toml, it finds torch already satisfied and doesn't try to pull the CUDA version.

**2. Add embedding-worker service to `docker/docker-compose.yml`:**

Add a new service definition after the task-executor service. Use the same image/build as task-executor but with a different command:

```yaml
  embedding-worker:
    # Uses the same image as task-executor (advance-rag)
    build:
      context: ../advance-rag
      dockerfile: ../docker/Dockerfile.task-executor  # Adjust path based on actual Dockerfile location
    command: python -m embedding_worker
    restart: unless-stopped
    depends_on:
      - valkey
    environment:
      # Database (read-only access for model config)
      - DB_HOST=${DB_HOST:-postgres}
      - DB_PORT=${DB_PORT:-5432}
      - DB_NAME=${DB_NAME:-b_knowledge}
      - DB_USER=${DB_USER:-postgres}
      - DB_PASSWORD=${DB_PASSWORD}
      # Redis/Valkey (Streams for embedding requests)
      - REDIS_HOST=${REDIS_HOST:-valkey}
      - REDIS_PORT=${REDIS_PORT:-6379}
      - REDIS_PASSWORD=${REDIS_PASSWORD:-}
      # Local Embedding config
      - LOCAL_EMBEDDING_ENABLE=${LOCAL_EMBEDDING_ENABLE:-false}
      - LOCAL_EMBEDDING_MODEL=${LOCAL_EMBEDDING_MODEL:-}
      - LOCAL_EMBEDDING_PATH=${LOCAL_EMBEDDING_PATH:-}
    volumes:
      # Persistent HuggingFace model cache -- survives container restarts (per D-06)
      - hf-cache:/root/.cache/huggingface
    networks:
      - kb-network
    # Each worker loads model independently (~600MB RAM per D-10)
    # Scale by running multiple replicas: docker compose up --scale embedding-worker=3
    deploy:
      resources:
        limits:
          memory: 2G
```

**3. Add hf-cache volume to the volumes section:**

In `docker/docker-compose.yml` (or `docker/docker-compose-base.yml` if volumes are defined there), add to the `volumes:` section at the bottom:

```yaml
volumes:
  hf-cache:
    driver: local
```

If volumes are split between the two compose files, add to whichever file defines the other named volumes (e.g., `app_logs`, `postgres-data`).

**4. Verify the network name:**

The service must be on the same network as valkey. Check the existing compose files for the network name (likely `kb-network` or `default`). Match the `networks` key accordingly.
