## Component Inventory

This phase adds 3 UI elements to 2 existing pages. No new pages or routes.

### 1. System Provider Badge (LLM Config page)

**Location:** Provider row on `/admin/llm-providers` page, inline with provider name.

| Property | Value |
|----------|-------|
| Component | `<Badge variant="info">` (existing `badge.tsx`) |
| Label | `System` (i18n key: `llmProviders.systemBadge`) |
| Placement | Inline after `factory_name / model_name` text, 8px gap |
| Condition | Render when `provider.is_system === true` |

**Interaction:**
- Badge is static, non-interactive
- Edit button: disabled with `cursor-not-allowed` and `opacity-50`
- Delete button: disabled with `cursor-not-allowed` and `opacity-50`
- Tooltip on disabled buttons: "Managed by LOCAL_EMBEDDING_MODEL environment variable" (i18n key: `llmProviders.systemManagedTooltip`)
- Admin CAN still click "Set as Default" for this provider (not disabled)

### 2. Re-embed Warning Banner (Dataset Settings Drawer)

**Location:** Inside `DatasetSettingsDrawer` > `GeneralSettingsForm`, above the form fields.

| Property | Value |
|----------|-------|
| Component | Custom `<Alert>` or inline div with warning styling |
| Background | `bg-yellow-50 dark:bg-yellow-950/30` |
| Border | `border border-yellow-200 dark:border-yellow-800` |
| Icon | `AlertTriangle` from Lucide, 16px, `text-yellow-600 dark:text-yellow-400` |
| Border radius | `rounded-md` (calc(var(--radius) - 2px)) |
| Padding | 12px (py-3 px-4) |
| Condition | Render when `dataset.embedding_dimension !== currentModel.dimension` |
| Layout | Icon left-aligned, text block right of icon (flex row, 8px gap), button below text |

**Copy:**
- Heading: "Embedding Model Mismatch" (i18n: `datasetSettings.reembed.mismatchTitle`)
- Body: "This dataset was embedded with a {oldDimension}d model. The current embedding model uses {newDimension}d vectors. Existing chunks will not match new queries until re-embedded." (i18n: `datasetSettings.reembed.mismatchBody`)
- CTA: "Re-embed All Chunks" button (i18n: `datasetSettings.reembed.action`)

### 3. Re-embed All Button

**Location:** Inside the warning banner (component 2 above).

| Property | Value |
|----------|-------|
| Component | `<Button variant="outline" size="sm">` |
| Label | "Re-embed All Chunks" |
| Placement | Below warning body text, left-aligned, 8px top margin |
| Loading state | Spinner + "Queuing..." text while mutation is in-flight |
| Disabled state | Disabled when re-embed task is already queued/running |
| Post-click | Toast success: "Re-embedding task queued. Track progress in the status bar." |

**Interaction flow:**
1. User opens Dataset Settings Drawer
2. Warning banner visible if dimension mismatch detected
3. User clicks "Re-embed All Chunks"
4. Confirmation dialog appears (see Destructive Confirmation below)
5. On confirm: mutation fires, button enters loading state
6. On success: toast notification, button text changes to "Re-embed Queued" (disabled)
7. Progress visible in existing `PipelineStatusBar` component

## State & Interaction Contract

### LLM Config Page — System Provider

| State | Visual |
|-------|--------|
| System provider present | Row shows `[System]` info badge, edit/delete buttons disabled with tooltip |
| System provider absent | No change to existing page (LOCAL_EMBEDDING_ENABLE=false) |
| System provider as default | Default model select shows the system provider selected, changeable |

### Dataset Settings — Re-embed Banner

| State | Visual |
|-------|--------|
| No dimension mismatch | Banner hidden, no change to existing form |
| Dimension mismatch detected | Yellow warning banner above form fields |
| Re-embed button idle | Outline button "Re-embed All Chunks" |
| Re-embed in-flight | Button shows spinner + "Queuing...", disabled |
| Re-embed queued | Button text "Re-embed Queued", disabled, warning banner remains |
| Re-embed error | Error toast, button returns to idle state |
