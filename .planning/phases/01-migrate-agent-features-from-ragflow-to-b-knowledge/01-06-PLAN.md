**agent-executor.service.ts — Graph orchestration engine:**

Singleton service that manages the full lifecycle of an agent run.

Core algorithm — Kahn's topological sort:
```
1. Parse DSL nodes and edges into adjacency list
2. Compute in-degree for each node
3. Initialize queue with all nodes having in-degree 0
4. While queue is not empty:
   a. Dequeue node
   b. Dispatch to Python via Redis if compute-heavy (generate, retrieval, code, tools)
     OR execute inline in Node.js if lightweight (switch, condition, merge, begin, answer)
   c. Wait for result (subscribe to Redis pub/sub)
   d. Store step result in agent_run_steps
   e. Pass output to downstream nodes (reduce their in-degree)
   f. Enqueue newly-ready nodes
5. When all nodes complete: mark run as 'completed'
6. On error: mark run as 'failed', store error in agent_run
```

Special handling:
- **Loop operator**: Track iteration count. Re-enqueue loop body nodes up to max_iterations. Treat loop-back edges separately from DAG edges.
- **Switch/Categorize**: Only enqueue the matching output branch edge.
- **Cycle detection**: Before execution, validate graph is DAG (ignoring Loop back-edges). Throw error on unexpected cycles.
- **Timeout**: Enforce max_execution_time from DSL settings. Cancel pending Redis tasks on timeout.

Methods:
- `startRun(agentId: string, input: string, tenantId: string, userId: string, triggerType: 'manual' | 'webhook' | 'embed'): Promise<string>` — Creates agent_run row, parses DSL, validates graph, starts execution loop, returns run_id
- `streamRun(runId: string, res: ExpressResponse): Promise<void>` — SSE streaming: subscribes to Redis pub/sub for run output, forwards delta events to client via res.write(), ends with [DONE]
- `cancelRun(runId: string, tenantId: string): Promise<void>` — Sets run status to cancelled, publishes cancel signal
- `getRunStatus(runId: string): Promise<AgentRunStatus>` — Returns current status from DB
- `private executeNode(node: AgentNodeDef, inputData: Record<string, unknown>, runId: string, step: AgentRunStep): Promise<Record<string, unknown>>` — Dispatches to Python or executes inline
- `private topologicalSort(dsl: AgentDSL): string[]` — Returns execution order, throws on cycles
- `private validateGraph(dsl: AgentDSL): void` — Checks for cycles, unreachable nodes, missing connections

**node_executor.py** — Node type execution dispatch:

```python
def execute_node(task: dict) -> dict:
    """Execute a single agent node based on its type.

    Args:
        task: Agent node task containing node_type, input_data, config, tenant_id.

    Returns:
        Dict with 'output_data' key containing the node's execution result,
        or 'error' key if execution failed.
    """
    node_type = task['node_type']
    input_data = task.get('input_data', {})
    config = task.get('config', {})
    tenant_id = task['tenant_id']

    # Dispatch to type-specific handler
    handler = NODE_HANDLERS.get(node_type)
    if not handler:
        return {'error': f'Unknown node type: {node_type}'}

    return handler(input_data, config, tenant_id)
```
