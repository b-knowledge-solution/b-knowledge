**agent-embed.service.ts:**

Singleton service for agent embed widget functionality. Follows chat-embed pattern.

Methods:
- `generateEmbedToken(agentId: string, tenantId: string): Promise<string>` — Creates embed token for agent widget (reuse existing embed token infrastructure from shared services or chat embed).
- `runFromEmbed(agentId: string, input: string, embedToken: string, res: ExpressResponse): Promise<void>` — Validates embed token, starts agent run with trigger_type='embed', streams output via SSE.
- `getAgentConfig(agentId: string, embedToken: string): Promise<{ name: string; avatar: string | null; description: string | null }>` — Returns minimal agent info for widget display.

**agent-embed.controller.ts:**

Handlers:
- `getConfig(req, res)` — GET /agents/embed/:agentId/config — Returns agent name/avatar for widget header
- `runEmbed(req, res)` — POST /agents/embed/:agentId/run — Accepts { input }, streams SSE response
- `getEmbedToken(req, res)` — POST /agents/:id/embed-token — Authenticated endpoint to generate embed token

**agent-embed.routes.ts:**

```typescript
const router = Router()
// Embed routes use embed token auth, not session auth
router.get('/:agentId/config', embedAuthMiddleware, agentEmbedController.getConfig)
router.post('/:agentId/run', embedAuthMiddleware, agentEmbedController.runEmbed)
export default router
```

**GenerateForm.tsx** — LLM Generate node configuration:
- LLM model selector (Select dropdown, populated from tenant's LLM providers)
- System prompt (Textarea, multi-line)
- Temperature slider (shadcn Slider, 0.0 - 2.0, step 0.1, default 0.7)
- Top-p slider (0.0 - 1.0, step 0.05, default 1.0)
- Max tokens input (number, default 2048)
- Frequency penalty slider (-2.0 to 2.0, default 0)
- "Message passthrough" toggle (passes upstream node output as user message)
- All fields update node config via canvasStore.updateNodeData on change
- JSDoc on component

**RetrievalForm.tsx** — RAG Retrieval node configuration:
- Dataset selector (multi-select from user's accessible datasets)
- Similarity threshold slider (0.0 - 1.0, default 0.3)
- Top-K input (number, 1-100, default 5)
- Retrieval method select: 'hybrid' | 'vector' | 'keyword'
- Reranking toggle (on/off)
- "Query source" select: upstream node output or custom query template
- JSDoc on component

**BeginForm.tsx** — Begin (start) node configuration:
- Input type: 'text' | 'file' | 'json' (Select)
- Variable definitions: list of agent variables with name, type, default
- Welcome message (Textarea)
- JSDoc on component

**SwitchForm.tsx** — Conditional switch node configuration:
- Condition list: dynamic add/remove condition rows
- Each condition: field name (Input), operator (Select: equals, contains, greater_than, less_than, regex), value (Input)
- Default branch label
- JSDoc on component

**CodeForm.tsx** — Code execution node configuration:
- Language selector: 'python' | 'javascript' | 'bash' (Select)
- Code editor (Textarea with monospace font, generous height)
- Timeout input (number, seconds, default 30)
- Input variable mapping: list of variable names to pass to code
- Output variable name (Input)
- JSDoc on component

All forms:
- Accept `NodeFormProps { nodeId: string; config: Record<string, unknown>; onUpdate: (data: Record<string, unknown>) => void }`
- Use native useState for form state (per FE conventions: no form libraries)
- Call onUpdate on field changes (which calls canvasStore.updateNodeData)
- Use shadcn Input, Select, Slider, Switch, Textarea, Button
- Support dark mode via Tailwind `dark:` classes
- JSDoc on all components
