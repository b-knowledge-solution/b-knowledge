**agent-webhook.service.ts:**

Singleton service for webhook-triggered agent execution.

Methods:
- `handleWebhook(agentId: string, body: Record<string, unknown>): Promise<{ run_id: string }>` — Validates agent exists and is published, extracts input from body, calls agentExecutorService.startRun with trigger_type='webhook', returns run_id.
- `validateWebhookPayload(body: unknown): { input: string }` — Extracts 'input' or 'message' or 'query' field from body. Throws 400 if no recognizable input field.

**agent-webhook.routes.ts:**

Public route (no auth required — webhook callers don't have user sessions):
```typescript
const router = Router()
// Webhook endpoint: no auth middleware, rate-limited separately
router.post('/:agentId', webhookLimiter, agentWebhookController.handleWebhook)
export default router
```

Apply a separate rate limiter: 100 requests per 15 minutes per IP.

**Update agent.controller.ts:**
Add `handleWebhook` method and template endpoints:
- `listTemplates(req, res)` — GET /agents/templates — Returns all templates (system + tenant-specific)

**Update app/routes.ts:**
- Add `import agentWebhookRoutes from '@/modules/agents/routes/agent-webhook.routes.js'`
- Mount: `router.use('/agents/webhook', agentWebhookRoutes)` BEFORE the authenticated `/agents` routes (webhook routes are unauthenticated)
- Add `router.get('/agents/templates', requireAuth, requireTenant, agentController.listTemplates)` inside the authenticated agent routes

**be/seeds/agent-templates.ts:**

Knex seed file with 24 pre-built agent workflow templates. Each template has: name, description, category, mode, dsl (complete JSONB graph), is_system=true, tenant_id=null.

Template categories and examples (derive from RAGFlow's 24 templates):
- **customer-support** (4): FAQ Bot, Complaint Handler, Ticket Router, Satisfaction Survey
- **data-processing** (4): Document Summarizer, Data Extractor, Report Generator, CSV Analyzer
- **research** (4): Research Assistant, Literature Review, Fact Checker, Trend Analyzer
- **content** (4): Blog Writer, Social Media Post, Email Drafter, Translation Helper
- **code** (4): Code Reviewer, Bug Reporter, Documentation Generator, API Tester
- **general** (4): Simple Chat, Multi-Step Reasoning, Task Planner, Decision Tree

Each template DSL should have a minimal but functional graph:
- At minimum: Begin node -> Generate node -> Answer node
- More complex templates add retrieval, conditions, loops as appropriate
- All nodes have sensible default configs

JSDoc on the seed function. Use `knex('agent_templates').insert(templates).onConflict('id').ignore()` to be idempotent.
