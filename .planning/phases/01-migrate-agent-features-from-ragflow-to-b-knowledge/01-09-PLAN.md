**agent-mcp.service.ts — MCP client integration:**

Singleton service managing MCP server connections.

```typescript
import { Client } from '@modelcontextprotocol/sdk/client/index.js'
import { StreamableHTTPClientTransport } from '@modelcontextprotocol/sdk/client/streamableHttp.js'
```

Methods:
- `connect(serverUrl: string): Promise<Client>` — Creates MCP client with HTTP transport, connects to server. Caches client by URL (connection pool at service level per RESEARCH open question 3).
- `listTools(serverUrl: string): Promise<McpTool[]>` — Lists available tools from MCP server.
- `callTool(serverUrl: string, toolName: string, args: Record<string, unknown>): Promise<unknown>` — Calls a tool on the MCP server with given arguments. Handles timeout (30s default).
- `disconnect(serverUrl: string): Promise<void>` — Closes connection and removes from pool.
- `disconnectAll(): Promise<void>` — Cleanup all connections (for graceful shutdown).

Error handling: Wrap all MCP calls in try/catch, return structured error responses. Log via Winston.

**agent-sandbox.service.ts — Docker sandbox for code execution:**

Singleton service managing ephemeral Docker containers.

```typescript
import Dockerode from 'dockerode'
```

Per Pitfall 5 in RESEARCH: DO NOT mount /var/run/docker.sock directly in production. Use docker-socket-proxy (tecnativa/docker-socket-proxy) in docker-compose for limited API surface.

Methods:
- `executeCode(code: string, language: 'python' | 'javascript' | 'bash', timeout: number = 30000): Promise<{ stdout: string; stderr: string; exitCode: number }>` — Creates ephemeral container, writes code to stdin, captures stdout/stderr, enforces timeout, removes container after execution.
- Container config: `{ Image: 'python:3.11-slim' (or node:22-slim), Cmd: [...], HostConfig: { Memory: 256 * 1024 * 1024 (256MB), NanoCPUs: 1e9 (1 CPU), NetworkMode: 'none' (no network), ReadonlyRootfs: true, AutoRemove: true } }`
- Timeout: Kill container if execution exceeds timeout. Return timeout error.
- Cleanup: Container auto-removes on exit. On error, manually remove to prevent orphans.
- Security: No volume mounts from sandbox. No network access. Read-only root filesystem.

**tools/tavily_tool.py** — Tavily web search tool:

```python
import httpx
from .base_tool import BaseTool

class TavilyTool(BaseTool):
    name = 'tavily'
    description = 'Web search via Tavily API'

    def execute(self, input_data, config, credentials=None):
        """Execute Tavily web search.

        Args:
            input_data: Must contain 'query' key with search query string.
            config: Optional 'max_results' (default 5), 'search_depth' ('basic' | 'advanced').
            credentials: Must contain 'api_key' for Tavily API.

        Returns:
            Dict with 'result' containing list of search results with title, url, content.
        """
        # Validate API key exists
        api_key = credentials.get('api_key') if credentials else None
        if not api_key:
            return {'error': 'Tavily API key not configured'}

        query = input_data.get('query', '')
        max_results = config.get('max_results', 5)
        search_depth = config.get('search_depth', 'basic')

        # Call Tavily search API
        response = httpx.post(
            'https://api.tavily.com/search',
            json={
                'api_key': api_key,
                'query': query,
                'max_results': max_results,
                'search_depth': search_depth,
            },
            timeout=30.0,
        )
        response.raise_for_status()
        data = response.json()

        return {
            'result': data.get('results', []),
            'answer': data.get('answer', ''),
        }
```

**tools/wikipedia_tool.py** — Wikipedia search tool:

```python
import httpx
from .base_tool import BaseTool

class WikipediaTool(BaseTool):
    name = 'wikipedia'
    description = 'Search and retrieve Wikipedia articles'

    def execute(self, input_data, config, credentials=None):
        """Search Wikipedia for articles matching the query.

        Args:
            input_data: Must contain 'query' key with search string.
            config: Optional 'language' (default 'en'), 'max_results' (default 3).

        Returns:
            Dict with 'result' containing article titles, snippets, and URLs.
        """
        query = input_data.get('query', '')
        language = config.get('language', 'en')
        max_results = config.get('max_results', 3)

        # Use Wikipedia API (no credentials needed)
        response = httpx.get(
            f'https://{language}.wikipedia.org/w/api.php',
            params={
                'action': 'query',
                'list': 'search',
                'srsearch': query,
                'srlimit': max_results,
                'format': 'json',
            },
            timeout=15.0,
        )
        response.raise_for_status()
        data = response.json()

        results = []
        for item in data.get('query', {}).get('search', []):
            results.append({
                'title': item['title'],
                'snippet': item.get('snippet', ''),
                'url': f'https://{language}.wikipedia.org/wiki/{item["title"].replace(" ", "_")}',
            })

        return {'result': results}
```
