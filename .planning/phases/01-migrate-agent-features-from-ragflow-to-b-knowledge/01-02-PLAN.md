Create the Zustand store for canvas state management. This store handles all ReactFlow node/edge state, selection, and undo/redo history.

**fe/src/features/agents/store/canvasStore.ts**

```typescript
import { create } from 'zustand'
import {
  type Node,
  type Edge,
  type OnNodesChange,
  type OnEdgesChange,
  type OnConnect,
  applyNodeChanges,
  applyEdgeChanges,
  addEdge,
} from '@xyflow/react'

interface CanvasHistoryEntry {
  nodes: Node[]
  edges: Edge[]
}

interface CanvasState {
  // Canvas data
  nodes: Node[]
  edges: Edge[]
  selectedNodeId: string | null
  isDirty: boolean

  // Undo/redo
  history: CanvasHistoryEntry[]
  historyIndex: number
  maxHistory: number

  // ReactFlow event handlers
  onNodesChange: OnNodesChange
  onEdgesChange: OnEdgesChange
  onConnect: OnConnect

  // Canvas mutations
  setNodes: (nodes: Node[]) => void
  setEdges: (edges: Edge[]) => void
  selectNode: (id: string | null) => void
  addNode: (node: Node) => void
  removeNode: (id: string) => void
  updateNodeData: (id: string, data: Record<string, unknown>) => void
  updateNodePosition: (id: string, position: { x: number; y: number }) => void

  // Persistence
  markClean: () => void
  loadFromDSL: (nodes: Node[], edges: Edge[]) => void

  // Undo/redo
  pushHistory: () => void
  undo: () => void
  redo: () => void
  canUndo: () => boolean
  canRedo: () => boolean
}
```

Implementation notes:
- `onNodesChange` calls `applyNodeChanges` and sets `isDirty: true`
- `onEdgesChange` calls `applyEdgeChanges` and sets `isDirty: true`
- `onConnect` calls `addEdge` and sets `isDirty: true`
- `pushHistory` snapshots current nodes/edges before mutations, capped at `maxHistory: 50`
- `undo` restores `history[historyIndex - 1]`, decrements index
- `redo` restores `history[historyIndex + 1]`, increments index
- `loadFromDSL` resets nodes/edges/history and marks clean
- Always use JSDoc on exported store and all methods
- `canUndo` returns `historyIndex > 0`, `canRedo` returns `historyIndex < history.length - 1`

IMPORTANT per Pitfall 2 in RESEARCH.md: consumers MUST use selectors like `useCanvasStore((s) => s.selectedNodeId)`, never destructure the full store. Add a JSDoc warning on the export.
