**agent-debug.service.ts:**

Singleton service extending the executor with debug capabilities.

Methods:
- `startDebugRun(agentId: string, input: string, tenantId: string, userId: string): Promise<string>` — Creates a debug run (trigger_type = 'manual'). Pauses before each node execution. Emits Socket.IO event `agent:debug:step` with status 'pending' for each node.
- `stepNext(runId: string, userId: string): Promise<void>` — Executes the next pending node. Emits `agent:debug:step` with status 'running' then 'completed'/'failed'. If node has a breakpoint, pauses after the next step.
- `continueRun(runId: string, userId: string): Promise<void>` — Runs all remaining nodes without pausing (except at breakpoints).
- `setBreakpoint(runId: string, nodeId: string): void` — Adds nodeId to breakpoint set for this run.
- `removeBreakpoint(runId: string, nodeId: string): void` — Removes nodeId from breakpoint set.
- `getStepDetails(runId: string, nodeId: string): Promise<AgentRunStep>` — Returns the step record with input/output data.

Socket.IO event shape:
```typescript
socketService.emitToUser(userId, 'agent:debug:step', {
  run_id: runId,
  node_id: nodeId,
  status: 'running' | 'completed' | 'failed' | 'skipped' | 'pending',
  input: nodeInput,     // Only sent on 'completed'
  output: nodeOutput,   // Only sent on 'completed'
  duration_ms: executionTime,
  error: errorMessage,  // Only sent on 'failed'
})
```

Internal state: Map of runId -> { pendingNodes: string[], breakpoints: Set<string>, currentIndex: number }. This state is in-memory (debug runs are ephemeral).

**agent-debug.controller.ts:**

Handlers:
- `startDebug(req, res)` — POST /agents/:id/debug — Starts debug run, returns { run_id }
- `stepNext(req, res)` — POST /agents/:id/debug/:runId/step — Execute next node
- `continueDebug(req, res)` — POST /agents/:id/debug/:runId/continue — Continue all
- `setBreakpoint(req, res)` — POST /agents/:id/debug/:runId/breakpoint — Body: { node_id }
- `removeBreakpoint(req, res)` — DELETE /agents/:id/debug/:runId/breakpoint/:nodeId
- `getStepDetails(req, res)` — GET /agents/:id/debug/:runId/steps/:nodeId

**Add debug routes to agent.routes.ts:**

```typescript
// Debug mode

router.delete('/:id/debug/:runId/breakpoint/:nodeId', agentDebugController.removeBreakpoint)

```

**useAgentStream.ts** — SSE streaming hook for agent run output:
- Creates EventSource connection to `/api/agents/${agentId}/run/${runId}/stream`
- Manages state: `output` (accumulated text), `isStreaming` (boolean), `error` (string | null)
- On `message` event: parse JSON, append delta to output
- On `[DONE]`: set isStreaming=false, close EventSource
- On error: set error message, close EventSource
- Cleanup: close EventSource on unmount
- Returns { output, isStreaming, error, startStream(runId) }
- Pattern matches existing useChatStream.ts using useState + useRef
- JSDoc on hook

**useAgentDebug.ts** — Socket.IO debug events hook:
- Subscribes to `agent:debug:step` event via existing useSocketEvent pattern
- Manages state: `steps` (Map of nodeId -> { status, input, output, duration_ms, error }), `isDebugActive` (boolean), `currentRunId` (string | null)
- On step event: update the node's status in steps map
- Methods: `startDebug(agentId)`, `stepNext()`, `continueRun()`, `toggleBreakpoint(nodeId)`
- `startDebug` calls POST /agents/:id/debug, stores runId
- `stepNext` calls POST /agents/:id/debug/:runId/step
- `continueRun` calls POST /agents/:id/debug/:runId/continue
- `toggleBreakpoint` calls POST or DELETE breakpoint endpoint
- Returns { steps, isDebugActive, startDebug, stepNext, continueRun, toggleBreakpoint, currentRunId }
- JSDoc on hook
