### Task 2: CLI wrapper

Created `fe/scripts/codemod-permissions.mjs` (~170 lines, zero runtime deps):

- Hand-rolled argv parser: `--dry-run`, `--files <pattern>`, `--verbose`.
- `resolveFiles()` strips trailing glob (everything from first `*`) to a base dir, then walks recursively. Single-file mode supported.
- Hard-coded exclusions baked into the walker AND single-file mode: `node_modules/`, `/dist/`, `fe/src/generated/`, plus extension allowlist `.ts | .tsx`.
- Per-file: read → `transformSource` → on `--dry-run` print BEFORE/AFTER markers, otherwise `writeFileSync`.
- Final summary line: `Processed N files, modified M files (K no-op) [DRY RUN]`.
- Top-level try/catch → `process.exit(1)` on any thrown error.

**Verification:**
```
$ node fe/scripts/codemod-permissions.mjs --dry-run --files 'fe/src/features/datasets/components/DocumentTable.tsx'
Processed 1 files, modified 0 files (1 no-op) [DRY RUN]
$ git diff --quiet fe/src/features/datasets/components/DocumentTable.tsx && echo "FILE UNCHANGED"
FILE UNCHANGED
```

The no-op result is correct: `DocumentTable.tsx` receives `isAdmin` as a destructured prop but does not contain a `<X isAdmin={...}/>` JSX usage at its own root, nor a `const isAdmin = user?.role === ...` declaration. Pattern A strips child usages (none here), Pattern B needs the const form (none here). The conservative behavior is to leave the file alone — Plan 4.4 will handle prop-receiver sites manually.

## Deviations from Plan

### [Rule 3 - Blocking] Indentation preservation

The plan's action body suggested `stmt.replaceWithText(\`${comment}\n${stmt.getText()}\`)` for prepending TODO comments. In practice ts-morph re-indents multi-line statement bodies when you `replaceWithText` an entire statement, which broke fixtures 02, 04, 05, 08. Resolution: buffer all prepend tasks as `{pos, comment}` pairs during the AST walk, then apply them via `sourceFile.insertText(pos, \`${comment}\n${indent}\`)` in reverse position order. Indent is computed from a source-text snapshot taken right before the apply phase. Documented in the lib via JSDoc on `getIndentFromText` and the inline comment above the apply loop. No semantic change vs the plan — only the implementation strategy changed.

### [Rule 3 - Blocking] Quote/semicolon style

ts-morph's default emitter uses double quotes + semicolons. The repo style is single quotes + no semicolons. Configured `manipulationSettings: { quoteKind: QuoteKind.Single }` for quotes; added a final-pass regex (`/^(import\s.+from\s+'[^']+');$/gm`) to strip trailing semicolons on inserted import lines. Scoped to import declarations only — does not touch arbitrary statements.

### [Rule 2 - Missing] Pattern C-2 (switch discriminant)

The plan's behavior list (Test 5) said "Switch-case on user.role → untouched + TODO comment above" but the action body's Pattern C only described `BinaryExpression` matching, which would not match a `SwitchStatement` whose cases are `case 'admin':` literals. Added a small Pattern C-2 block that finds `SwitchStatement` nodes whose discriminant text matches `user?.role` / `user.role` and prepends the same `TODO_REVIEW` marker. This is correctness-required to make fixture 05 testable.

### [Rule 3 - Blocking] Test runner registration

`fe/vitest.shared.ts` uses an explicit allowlist (`unitTestFiles`) rather than a glob include — the new test file `scripts/__tests__/codemod-permissions.test.ts` was invisible until added to that array. Added it as the 12th entry. No semantic change.

### Note: `npm run test:run` (full) caveat

The plan's verification command is `cd fe && npm run test:run -- scripts/__tests__/codemod-permissions.test.ts`. `test:run` chains `test:run:unit && test:run:ui`; the ui suite's include is `tests/**/*.test.{ts,tsx}` and excludes our path, so passing the filter to the full chain causes the ui suite to error with "No test files found". The unit suite alone runs cleanly: `cd fe && npm run test:run:unit` → 11 files / 150 passing tests including all 8 codemod fixtures. Direct vitest invocation also works:
```
cd fe && npx vitest run -c vitest.unit.config.ts scripts/__tests__/codemod-permissions.test.ts
```
Recording this as a note rather than a fix because the workaround (run unit-only) is clean and changing the ui suite is out of scope for Plan 4.3.
