The system uses a microservice architecture where each service communicates through well-defined APIs. Data flows from ingestion through processing to storage.

Configuration management is handled centrally through environment variables and mounted config files.

Chunk #3
Documents are split into chunks using recursive character splitting with configurable chunk size and overlap parameters. Each chunk is then embedded using the configured embedding model and stored in the vector database for similarity search. The chunking strategy can be customized per document type — for example, code files use syntax-aware splitting while PDFs use page-boundary-aware splitting.

After embedding, chunks are indexed in OpenSearch with both vector and keyword fields to enable hybrid search capabilities.
