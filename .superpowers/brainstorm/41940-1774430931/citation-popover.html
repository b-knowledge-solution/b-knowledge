Documents are split into chunks using recursive character splitting with configurable chunk size and overlap parameters. Each chunk is then embedded using the configured embedding model and stored in the vector database for similarity search. The chunking strategy can be customized per document type — for example, code files use syntax-aware splitting while PDFs use page-boundary-aware splitting. Overlapping windows ensure that context is not lost at chunk boundaries, which is critical for retrieval accuracy.
